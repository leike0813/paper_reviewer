# 通用科研论文写作框架

## 1. 适用范围

本指南适用于各学科科研论文写作的通用底层框架，尤其适合用于 Agent Skill 在以下任务中作为共同知识基座：

- 辅助用户构思科研论文；
- 判断论文结构是否完整；
- 提取和重构研究问题；
- 检查论证链是否成立；
- 改写 Introduction、Literature Review、Method、Results、Discussion、Conclusion；
- 辅助审稿意见回复；
- 判断论文贡献是否清晰；
- 对已有初稿进行诊断、重组和润色。

本指南不针对单一学科，而关注科研论文作为一种**学术论证文本**的基本机制。无论论文采用实验、调查、建模、田野、文本分析、理论论证、系统设计、案例研究、语料分析、历史解释或规范分析等方法，其核心任务通常都包括：

1. 提出一个值得研究的问题；
2. 说明已有研究为什么还不够；
3. 提供可信的研究路径；
4. 给出证据、分析或论证；
5. 声明相对于既有知识的贡献；
6. 说明贡献的意义、边界和后续可能。

本指南可作为其他具体范式文档的总纲，例如：

- 实证量化论文写作指南；
- 质性研究论文写作指南；
- 理论论文写作指南；
- 综述论文写作指南；
- 工程系统论文写作指南；
- 计算实验论文写作指南；
- 人文学科解释型论文写作指南；
- 规范性、政策性或伦理分析论文写作指南。

---

## 2. 不适用范围

本指南不应被机械套用于以下文本：

1. **课程论文或读书报告**  
   这类文本可以总结和评述文献，但未必需要形成原创知识贡献。

2. **项目总结报告**  
   项目报告重点在任务完成、流程说明和成果汇报；科研论文重点在可辩护的知识贡献。

3. **政策白皮书或咨询报告**  
   这类文本通常面向决策者，强调建议和可执行方案；科研论文需要更严格地交代理论基础、证据链和研究边界。

4. **科普文章**  
   科普文章重在解释知识；科研论文重在生产和论证知识。

5. **纯技术文档或用户手册**  
   技术文档说明如何使用系统；科研论文需要解释为什么该系统、方法或发现具有学术意义。

6. **没有研究问题的资料汇编**  
   单纯罗列事实、案例、数据或文献，不构成科研论文。

7. **只表达观点的评论文章**  
   学术论文可以有立场，但立场必须经过概念界定、文献定位、证据支持和论证限定。

## 3. 知识生产逻辑

科研论文不是“把研究过程写出来”，而是把研究过程组织成一个可被审稿人检验的知识主张。其核心逻辑是：

> 从一个有意义的问题出发，在已有研究的基础上定位缺口，通过合适的方法、材料、理论或论证生成证据，最终提出一个相对清晰、有限但可信的贡献。

### 3.1 科研论文的基本任务

| 基本任务 | 写作中的表现 | Agent 判断重点 |
|---|---|---|
| 提出问题 | 说明研究对象、现象、矛盾、异常或理论张力 | 是否是研究问题，而不是泛泛主题 |
| 定位缺口 | 指出已有研究未解决、未充分解释或存在争议之处 | 缺口是否真实、具体、可研究 |
| 构建论证 | 说明本文如何从问题走向结论 | 论证链是否连续 |
| 提供证据 | 数据、文本、案例、实验、模型、材料、逻辑证明等 | 证据是否足以支撑主张 |
| 声明贡献 | 说明本文增加了什么新知识 | 贡献是否超出“做了一个研究” |
| 说明边界 | 明确适用条件、限制和未来工作 | 是否避免过度声称 |

### 3.2 研究问题、研究目标、研究假设、研究命题、核心论点的区别

Agent 辅助写作时必须区分以下概念，避免混用。

#### 研究主题

研究主题是论文关注的宽泛领域。

示例：

> 数字平台治理、癌症免疫治疗、跨文化翻译、城市热岛效应、算法公平。

主题本身不是研究问题。

错误写法：

> 本文研究人工智能伦理。

这只是主题，不是问题。

较好写法：

> 本文研究生成式人工智能在高校写作评估中的责任归属问题。

#### 研究问题

研究问题是论文要回答的具体疑问，通常可以转化为“为什么”“如何”“在何种条件下”“是否”“多大程度上”等形式。

示例：

> 为什么某些平台治理措施降低了违规内容数量，却未能改善用户对公平性的感知？

质量判断：

- 是否具体；
- 是否可回答；
- 是否与文献缺口相关；
- 是否决定了方法、材料和论证结构；
- 是否能在结论中得到回应。

#### 研究目标

研究目标说明本文要完成什么任务。

示例：

> 本文旨在解释平台治理透明度如何影响用户对治理公平性的判断。

研究目标比研究问题更偏行动导向。  
研究问题问“要回答什么”，研究目标说“要完成什么”。

#### 研究假设

研究假设通常用于实证检验，尤其常见于量化研究、实验研究和模型检验。

示例：

> H1：平台治理规则的可解释性越高，用户感知公平性越高。

质量判断：

- 是否可检验；
- 是否来源于理论或文献；
- 是否具有方向性或明确关系；
- 是否与变量、数据和方法对应。

#### 研究命题

研究命题常见于理论论文、概念论文、质性研究或机制分析中。它不一定立即以统计方式检验，但需要被理论、案例或逻辑论证支持。

示例：

> 命题1：在高不确定性环境中，组织更倾向于采用可逆性治理机制以降低制度锁定风险。

质量判断：

- 是否明确表达概念之间的关系；
- 是否具有理论解释力；
- 是否可被后续研究检验、修正或应用。

#### 核心论点

核心论点是整篇论文最终要说服读者接受的中心主张。

示例：

> 本文认为，平台治理中的透明度并不必然提升公平性感知；只有当透明度同时伴随可申诉性时，用户才会将其解释为程序公平。

质量判断：

- 是否能概括全文贡献；
- 是否比研究问题更接近答案；
- 是否有证据或论证支撑；
- 是否避免空泛表述。

### 3.3 从“主题”到“贡献”的转换链

一篇论文的知识生产链通常应能被压缩为以下形式：

```text
研究主题
→ 具体问题
→ 文献缺口
→ 研究设计 / 理论路径
→ 证据 / 分析 / 论证
→ 发现 / 主张
→ 贡献
→ 边界与意义
```

Agent 检查论文时，应尝试为用户生成一条“单句论证链”：

> 由于已有研究主要关注 A，却较少解释 B，本文基于 C 材料/方法/理论，分析 D，发现/论证 E，从而为 F 文献或实践问题提供 G 贡献。

如果无法生成这条链，通常说明论文存在以下问题之一：

- 研究问题不清；
- 文献缺口不真实；
- 方法与问题不匹配；
- 结果只是描述，没有上升为发现；
- 贡献没有相对于已有研究表达；
- 讨论没有回到问题。

### 3.4 Claim-Evidence-Warrant 逻辑

科研论文中的每一个重要判断都应包含三部分：

| 元素 | 含义 | 示例 |
|---|---|---|
| Claim | 主张 | 平台规则透明度会影响用户公平性感知 |
| Evidence | 证据 | 调查数据、访谈材料、实验结果、文本分析、模型输出 |
| Warrant | 论证桥梁 | 透明度使用户能够理解处罚依据，因此影响程序公平判断 |

常见失败模式：

- 只有 Claim，没有 Evidence；
- 有 Evidence，但没有解释它为什么支持 Claim；
- Evidence 与 Claim 不匹配；
- Claim 超出 Evidence 能支持的范围；
- Warrant 依赖未经说明的假设。

Agent 判断规则：

```text
每出现一个强主张，都检查：
1. 证据在哪里？
2. 证据是否直接相关？
3. 作者是否解释了证据为什么支持该主张？
4. 是否存在替代解释？
5. 作者是否限定了主张适用范围？
```

## 4. 典型论文结构

科研论文的结构不是固定模板，但多数论文都需要完成类似的功能。常见结构如下：

```text
Title
Abstract
Keywords

1. Introduction
2. Literature Review / Related Work / Theoretical Background
3. Method / Materials / Data / Corpus / Theory / Research Design
4. Results / Analysis / Findings / Argument
5. Discussion
6. Conclusion

References
Appendices / Supplementary Materials
```

### 4.1 章节功能总览

| 章节 | 核心功能 | 不应承担的功能 |
|---|---|---|
| Title | 准确压缩研究对象、问题、方法或贡献 | 夸张宣传 |
| Abstract | 高密度呈现问题、方法、结果和贡献 | 写成背景介绍 |
| Keywords | 帮助检索和定位领域 | 堆砌宽泛词 |
| Introduction | 建立问题重要性、缺口和贡献 | 详细综述所有文献 |
| Literature Review | 构建问题空间和贡献位置 | 罗列文献摘要 |
| Method / Data / Theory | 说明知识如何被生产 | 写成流水账 |
| Results / Analysis | 呈现发现、分析或论证主体 | 只堆数据或材料 |
| Discussion | 解释结果意义、比较既有研究、说明边界 | 重复结果 |
| Conclusion | 压缩贡献，回到问题，指出影响 | 引入大量新证据 |
| References | 支撑学术定位和论证来源 | 装饰性引用 |
| Appendix | 放置辅助材料 | 承担主论证 |

### 4.2 通用结构判断规则

Agent 可用以下问题判断结构是否有效：

1. 读完 Introduction，是否知道论文要解决什么问题？
2. 读完 Literature Review，是否知道本文站在什么文献争议中？
3. 读完 Method / Data / Theory，是否知道作者如何生产证据？
4. 读完 Results / Analysis，是否知道作者发现或证明了什么？
5. 读完 Discussion，是否知道这些发现意味着什么？
6. 读完 Conclusion，是否能复述本文贡献和边界？
7. 每一章是否承担了不同功能，而不是互相重复？
8. 章节顺序是否符合“问题—路径—证据—贡献”的逻辑？

## 5. 各章节写作指南

## 5.1 标题 Title

### 写作目标

标题应准确传达论文的研究对象、核心问题、方法视角或主要贡献，使目标读者能够快速判断论文是否相关。

### 应包含内容

根据论文类型，标题可以包含以下元素中的两到三个：

- 研究对象；
- 核心现象；
- 方法或材料；
- 理论视角；
- 关键变量或概念；
- 研究场景；
- 主要关系或机制。

### 常见标题类型

| 类型 | 结构 | 示例形式 |
|---|---|---|
| 对象 + 问题 | 关于 X 的 Y 问题研究 | 数字平台劳动者的算法控制感知研究 |
| 问题 + 方法 | 基于 X 的 Y 分析 | 基于多案例比较的城市韧性治理机制分析 |
| 发现式标题 | X 如何影响 Y | 治理透明度如何影响用户公平性感知 |
| 理论式标题 | 重新理解 X | 重新理解组织韧性：一种过程视角 |
| 主副标题 | 概念性主标题 + 解释性副标题 | 看得见的规则：平台治理透明度与程序公平感知 |

### 常见错误

- 标题过大，如“人工智能伦理研究”；
- 标题只写对象，不写问题；
- 使用宣传式语言，如“革命性”“颠覆性”“全新突破”；
- 标题包含过多概念；
- 标题与正文实际贡献不一致；
- 标题承诺因果关系，但正文只有相关性分析。

### Agent 质量判断

标题应通过以下测试：

```text
1. 是否能看出研究对象？
2. 是否能看出问题或关系？
3. 是否过度承诺？
4. 是否与论文方法和结果一致？
5. 是否比“主题名词”更具体？
```

## 5.2 摘要 Abstract

### 写作目标

摘要是整篇论文的压缩版，应让读者在不阅读全文的情况下理解：

- 研究背景；
- 研究问题；
- 方法或材料；
- 核心发现；
- 贡献和意义。

### 推荐结构

通用摘要可采用五句式：

```text
1. 背景句：指出研究领域或现象的重要性。
2. 缺口句：说明已有研究不足或争议。
3. 方法句：说明本文使用的数据、材料、方法或理论路径。
4. 发现句：概括最重要的结果、发现或论点。
5. 贡献句：说明本文对理论、方法、实证、实践或规范讨论的贡献。
```

### 示例骨架

> 随着 X 的发展，Y 问题受到越来越多关注。已有研究主要从 A 角度解释该问题，但对 B 的机制仍缺乏充分说明。本文基于 C 数据/材料/案例/理论，采用 D 方法，分析 E。研究发现，F 并非简单导致 G，而是通过 H 条件产生差异化影响。本文的贡献在于揭示了 I 机制，并为 J 文献/实践提供了新的解释框架。

### 常见错误

- 摘要只有背景，没有研究结果；
- 摘要写成引言第一段；
- 摘要中使用“本文具有重要意义”但不说明意义是什么；
- 方法描述模糊，如“采用科学方法进行分析”；
- 发现过于笼统，如“结果表明二者存在关系”；
- 摘要中出现正文没有展开的贡献；
- 摘要过度宣传，缺少限定条件。

### Agent 质量判断

Agent 应检查摘要是否回答：

```text
1. 研究了什么问题？
2. 为什么这个问题值得研究？
3. 用什么方法、材料或理论研究？
4. 得到了什么具体结果或主张？
5. 相比已有研究增加了什么？
6. 是否避免空泛价值判断？
```

## 5.3 关键词 Keywords

### 写作目标

关键词用于帮助论文被检索、分类和定位学术共同体。

### 应包含内容

关键词通常包括：

- 核心研究对象；
- 核心概念；
- 方法或理论；
- 研究场景；
- 领域术语。

### 常见错误

- 使用过宽关键词，如“社会”“文化”“技术”；
- 关键词与标题完全重复但没有扩展检索入口；
- 使用正文中很少出现的词；
- 堆砌热门词；
- 使用未定义缩写。

### Agent 质量判断

关键词应满足：

```text
1. 是否与论文核心问题直接相关？
2. 是否覆盖对象、理论/方法、场景或领域？
3. 是否利于目标读者检索？
4. 是否避免过宽或过窄？
```

## 5.4 Introduction 引言

### 写作目标

Introduction 的任务不是展示所有背景知识，而是把读者带入论文的问题、缺口和贡献。它应完成五项功能：

1. 建立研究背景；
2. 提出具体问题；
3. 定位文献缺口；
4. 说明本文如何回应缺口；
5. 预告贡献和文章结构。

### 推荐组织方式

可采用“漏斗式”结构：

```text
宽背景
→ 具体现象或理论问题
→ 已有研究做了什么
→ 仍然缺少什么
→ 本文研究问题
→ 本文方法/材料/路径
→ 本文发现或论点
→ 本文贡献
→ 文章结构
```

### 引言中的关键句型

#### 背景句

> X 已成为 Y 领域的重要问题。  
> 近年来，A 的变化使 B 问题更加突出。  
> 在 C 背景下，理解 D 对于解释 E 具有重要意义。

#### 缺口句

> 尽管已有研究从 A 和 B 角度进行了讨论，但对 C 的解释仍不充分。  
> 现有研究多关注 X，却较少分析 Y。  
> 关于 A 是否导致 B，已有研究仍存在分歧。

#### 研究问题句

> 因此，本文关注以下问题：X 如何影响 Y？  
> 本文试图回答：在何种条件下，A 会转化为 B？  
> 本文的核心问题是：为什么 X 在相似条件下产生了不同结果？

#### 贡献句

> 本文的贡献主要体现在三个方面。第一，……第二，……第三，……  
> 与既有研究相比，本文将 A 与 B 结合起来，揭示了 C 机制。  
> 本文通过引入 X 视角，修正了 Y 文献中关于 Z 的假设。

### 常见错误

- 引言写成宏大背景介绍；
- 第一段过于宽泛，如“自古以来”“随着社会发展”；
- 缺口只是声称“研究较少”，没有说明为什么重要；
- 研究问题没有明确出现；
- 贡献写成“丰富了研究”“具有理论和实践意义”，但没有具体内容；
- 引言承诺的内容正文没有兑现；
- 过早进入方法细节；
- 文献综述过长，挤压问题提出。

### Agent 质量判断

Agent 应检查 Introduction 是否满足：

```text
1. 是否在前几段内明确研究对象？
2. 是否从背景推进到具体问题？
3. 是否说明已有研究的不足，而不是简单说“较少研究”？
4. 是否明确写出研究问题或核心目标？
5. 是否说明本文如何研究该问题？
6. 是否声明具体贡献？
7. 是否与后文章节一致？
```

---

## 5.5 Literature Review / Related Work 文献综述

### 写作目标

文献综述不是“谁说了什么”的清单，而是为本文搭建学术位置。它要回答：

- 这个问题属于哪些研究传统；
- 已有研究如何解释该问题；
- 这些解释之间有什么差异、争议或盲区；
- 本文为什么需要介入；
- 本文将如何推进现有讨论。

### 应包含内容

文献综述通常包括：

1. 核心概念界定；
2. 相关研究流派或主题；
3. 已有解释路径；
4. 研究争议；
5. 未解决问题；
6. 本文定位。

### 推荐组织方式

#### 方式一：主题式综述

适合多数论文。

```text
主题一：已有研究如何解释 A
主题二：已有研究如何解释 B
主题三：A 与 B 的关系研究
小结：现有研究不足与本文切入点
```

#### 方式二：争议式综述

适合理论争议明显的论文。

```text
观点一：X 导致 Y
观点二：X 不必然导致 Y
观点三：关系取决于 Z 条件
本文立场：引入 W 机制解释分歧
```

#### 方式三：演进式综述

适合概念、理论或领域发展类论文。

```text
早期研究：关注 A
中期研究：转向 B
近期研究：强调 C
仍未解决：D
本文推进：E
```

#### 方式四：方法式综述

适合方法论文或计算研究。

```text
传统方法：优势与局限
改进方法：解决了什么问题
现有瓶颈：仍存在什么不足
本文方法：如何改进
```

### 常见错误

- 按作者逐篇罗列；
- 文献之间没有关系；
- 只总结观点，不评价不足；
- 只找“没人研究过”作为缺口；
- 文献综述与研究问题脱节；
- 只引用支持自己的文献，不处理相反观点；
- 综述结束后没有明确本文位置；
- 概念使用不稳定。

### Agent 质量判断

Agent 应检查：

```text
1. 文献是否围绕研究问题组织？
2. 是否形成主题、争议或解释路径？
3. 是否说明已有研究的贡献和不足？
4. 是否避免逐篇摘要式写法？
5. 是否自然推出本文缺口？
6. 是否界定了核心概念？
7. 是否处理了重要反方观点或替代解释？
```

---

## 5.6 Method / Materials / Data / Corpus / Theory 方法、材料、数据、语料或理论路径

### 写作目标

这一部分说明论文的知识是如何被生产出来的。不同学科名称不同，但核心功能相同：

> 让读者判断本文的证据来源、分析路径和结论可信度。

### 不同形态

| 章节名称 | 常见领域 | 核心功能 |
|---|---|---|
| Method | 实验、调查、计算、社会科学 | 说明研究设计、变量、程序、分析方法 |
| Materials | 实验科学、工程、医学等 | 说明实验材料、样本、设备、条件 |
| Data | 量化研究、计算研究、社会科学 | 说明数据来源、采集、处理和质量 |
| Corpus | 语言学、文本研究、数字人文 | 说明语料来源、规模、选择标准和标注 |
| Case Selection | 案例研究、比较研究 | 说明为什么选择这些案例 |
| Fieldwork | 人类学、社会学、地理等 | 说明田野地点、对象、过程和伦理 |
| Theoretical Framework | 理论论文、人文学科 | 说明概念工具、理论假设和分析路径 |
| Model | 经济学、工程、计算、物理 | 说明模型结构、参数、假设和验证方式 |

### 应包含内容

根据论文类型，通常需要说明：

- 研究设计；
- 数据或材料来源；
- 样本、案例或语料选择标准；
- 数据采集过程；
- 数据处理过程；
- 变量、指标、概念或编码方案；
- 分析方法；
- 可靠性、有效性或可信度保障；
- 伦理说明；
- 局限性。

### 常见错误

- 只说“采用定量方法”“采用质性方法”，没有细节；
- 方法描述无法复现或无法理解；
- 没有说明样本、案例或语料为什么合适；
- 方法与研究问题不匹配；
- 变量或概念没有定义；
- 数据清洗、编码、筛选过程不透明；
- 只描述工具，不说明分析逻辑；
- 把结果提前写进方法部分；
- 忽略伦理、偏差和限制。

### Agent 质量判断

Agent 应检查：

```text
1. 读者是否知道证据从哪里来？
2. 读者是否知道材料为何适合回答研究问题？
3. 研究设计是否与研究问题匹配？
4. 分析步骤是否足够透明？
5. 样本、案例、语料或数据选择是否有理由？
6. 是否说明质量控制方式？
7. 是否承认方法限制？
```

## 5.7 Results / Findings / Analysis / Argument 结果、发现、分析或论证主体

### 写作目标

主体部分负责展示论文真正生产出的知识。不同范式中名称不同：

- 实验论文写 Results；
- 质性论文写 Findings；
- 人文学科论文写 Analysis；
- 理论论文写 Argument；
- 工程论文写 Evaluation；
- 计算论文写 Experiments / Results；
- 规范论文写 Normative Analysis。

其共同目标是：

> 以有组织的方式呈现证据、分析和中间结论，使读者看到核心主张如何被支持。

### 结果、分析、论证的区别

| 类型 | 重点 | 示例 |
|---|---|---|
| Results | 呈现数据结果 | 模型性能、统计关系、实验差异 |
| Findings | 提炼经验发现 | 访谈主题、案例机制、观察模式 |
| Analysis | 解释材料意义 | 文本解读、机制分析、比较分析 |
| Argument | 推进理论主张 | 概念推演、规范论证、逻辑证明 |

### 推荐组织原则

主体部分不应按“研究过程顺序”写，而应按“论证需要”组织。常见方式包括：

#### 按研究问题组织

```text
RQ1 的结果
RQ2 的结果
RQ3 的结果
```

适合多研究问题论文。

#### 按假设组织

```text
H1 检验
H2 检验
H3 检验
补充分析
```

适合假设检验型论文。

#### 按机制组织

```text
机制一：A 如何触发 B
机制二：B 如何转化为 C
机制三：C 如何导致 D
```

适合解释型论文。

#### 按主题组织

```text
主题一
主题二
主题三
主题之间的关系
```

适合质性研究和文本分析。

#### 按比较维度组织

```text
案例一与案例二的相似性
案例一与案例二的差异
差异背后的条件
```

适合比较研究。

#### 按论证步骤组织

```text
概念界定
前提说明
推论一
推论二
反驳替代观点
综合命题
```

适合理论论文和规范论文。

### 常见错误

- 只呈现数据，不解释意义；
- 只解释观点，不提供证据；
- 表格、图像和文字重复；
- 结果顺序与研究问题不一致；
- 把所有发现等量呈现，没有主次；
- 选择性报告有利结果；
- 过早在结果部分做宏大讨论；
- 结论超出结果；
- 没有处理异常结果或反例。

### Agent 质量判断

Agent 应检查：

```text
1. 主体部分是否围绕研究问题展开？
2. 每个小节是否有明确小结论？
3. 证据是否支持对应主张？
4. 是否解释了结果的含义？
5. 是否处理了反例、异常或替代解释？
6. 图表是否承担信息功能，而不是装饰？
7. 结果是否为 Discussion 提供基础？
```

## 5.8 Discussion 讨论

### 写作目标

Discussion 的功能是把结果、发现或论证提升为学术贡献。它回答：

- 这些结果意味着什么？
- 它们如何回应研究问题？
- 它们与已有研究一致还是不同？
- 它们改变了我们对什么问题的理解？
- 它们适用于哪些情境，不适用于哪些情境？
- 它们有什么理论、方法、实践或规范意义？

### 推荐组织方式

```text
1. 简要回扣核心发现
2. 解释发现背后的含义
3. 与已有研究对话
4. 提炼理论/方法/实践贡献
5. 讨论边界和限制
6. 指出未来研究方向
```

### 常见讨论类型

| 类型 | 功能 |
|---|---|
| 解释型讨论 | 解释为什么出现该结果 |
| 对话型讨论 | 与已有研究比较，说明一致、扩展或修正 |
| 贡献型讨论 | 把发现上升为理论、方法或实践贡献 |
| 边界型讨论 | 说明适用条件和限制 |
| 反思型讨论 | 处理异常结果、方法限制或伦理问题 |

### 常见错误

- 讨论只是重复结果；
- 讨论突然引入新材料；
- 只说“具有重要意义”，不说明意义；
- 不回应文献综述中的争议；
- 不讨论与预期不一致的结果；
- 过度推广；
- 局限性写得敷衍；
- 未来研究方向与本文无关。

### Agent 质量判断

Agent 应检查：

```text
1. Discussion 是否回到研究问题？
2. 是否解释结果，而不只是重复结果？
3. 是否与文献综述中的关键文献对话？
4. 是否明确贡献类型？
5. 是否说明边界条件？
6. 是否承认限制但不自我否定？
7. 是否提出合理的未来研究方向？
```

---

## 5.9 Conclusion 结论

### 写作目标

Conclusion 应压缩整篇论文的核心贡献，使读者清楚记住本文回答了什么问题、增加了什么知识、还有什么边界。

### 应包含内容

通用结论可包括：

1. 回到研究问题；
2. 概括研究路径；
3. 总结核心发现或论点；
4. 强调贡献；
5. 说明影响；
6. 指出限制与未来方向。

### 推荐结构

```text
本文围绕 X 问题展开，旨在解释/检验/分析 Y。
基于 Z 方法/数据/材料/理论，本文发现/论证 A。
这一发现/论证表明 B。
本文的贡献在于 C。
当然，本文仍存在 D 限制，未来研究可进一步探讨 E。
```

### 常见错误

- 只是重复摘要；
- 引入新证据或新文献；
- 用口号式语言结束；
- 不回应研究问题；
- 贡献表述空泛；
- 局限性与未来研究写成例行套话；
- 结论比正文声称更强。

### Agent 质量判断

Agent 应检查：

```text
1. 是否明确回答了研究问题？
2. 是否总结了最重要的发现或论点？
3. 是否说明了贡献，而不是重复过程？
4. 是否避免引入新证据？
5. 是否有合理边界？
6. 是否与 Introduction 中的承诺闭合？
```

---

## 5.10 References 参考文献

### 写作目标

参考文献不是形式附属物，而是论文知识位置的证据。它说明本文进入了哪个学术共同体、回应了哪些研究、依赖了哪些理论和方法。

### 应包含内容

参考文献应覆盖：

- 领域经典文献；
- 近期关键研究；
- 理论来源；
- 方法来源；
- 数据或材料来源；
- 与本文立场相反或不同的研究；
- 直接支撑核心判断的文献。

### 常见错误

- 只引用少数综述或教材；
- 忽视近年重要研究；
- 引用与正文无关；
- 文献堆砌；
- 用二手引用替代原始来源；
- 引用格式不一致；
- 文献综述中引用了文献，但讨论中不再对话；
- 只引用支持性文献，不引用争议性文献。

### Agent 质量判断

```text
1. 关键概念是否有来源？
2. 主要理论是否有代表性文献？
3. 方法是否有出处？
4. 是否包含近期研究？
5. 是否处理相反观点？
6. 引用是否服务论证，而非装饰？
```

## 5.11 图表、公式、脚注与附录

### 图表

#### 写作目标

图表用于压缩复杂信息，展示文字难以高效表达的结构、关系、趋势或比较。

#### 好图表应满足

- 有清晰标题；
- 变量、单位、维度明确；
- 读者不看正文也能理解基本信息；
- 图表与正文互相补充；
- 正文解释图表中的关键模式，而不是逐项复述。

#### 常见错误

- 图表无标题或标题过于模糊；
- 图表与正文重复；
- 图表过于复杂；
- 缺少单位、样本量、注释；
- 只放图表，不解释其意义；
- 图表支持不了正文主张。

### 公式

#### 写作目标

公式用于精确表达模型、关系、定义或推导。

#### 常见错误

- 公式符号未定义；
- 公式与文本脱节；
- 推导跳步过大；
- 公式看似复杂但不服务论证；
- 没有解释参数含义。

### 引用

#### 写作目标

引用用于：

- 支撑事实判断；
- 标明理论来源；
- 进入学术对话；
- 区分本文贡献和已有观点。

#### 常见错误

- 每句话都引用但没有综合；
- 引用替代论证；
- 断章取义；
- 只引用权威名词，不说明相关性；
- 引文堆积在段落末尾。

### 脚注

#### 写作目标

脚注适合放置补充说明、术语说明、材料来源或不宜打断正文的限定。

#### 常见错误

- 用脚注承载核心论证；
- 脚注过长；
- 脚注过多影响阅读；
- 正文依赖脚注才能理解。

### 附录

#### 写作目标

附录用于放置支撑主文但不适合放入正文的材料，例如：

- 问卷；
- 访谈提纲；
- 编码表；
- 额外模型；
- 稳健性检验；
- 语料样例；
- 补充证明；
- 数据处理细节。

#### 常见错误

- 主论证依赖附录才能成立；
- 附录没有在正文中引用；
- 附录材料无说明；
- 附录成为“垃圾箱”。

## 6. 常见贡献类型

科研论文的贡献不一定只有一种。Agent 应帮助用户识别并准确表达贡献类型。

### 6.1 理论贡献

通过提出、修正、扩展或整合理论来增加知识。

常见形式：

- 提出新概念；
- 修正已有理论假设；
- 扩展理论适用范围；
- 整合两个原本分离的理论传统；
- 解释已有理论无法解释的现象。

弱表达：

> 本文丰富了相关理论。

强表达：

> 本文通过引入“可申诉性”概念，修正了透明度研究中“信息可见即提升公平感”的隐含假设。

### 6.2 方法贡献

通过新的研究方法、测量方式、模型、工具、数据处理路径或分析框架产生贡献。

常见形式：

- 提出新算法；
- 改进测量指标；
- 构建新数据集；
- 改进编码框架；
- 提供可复用研究流程；
- 证明某方法适用于新问题。

弱表达：

> 本文采用了新方法。

强表达：

> 本文提出一种结合语义嵌入与人工校验的政策文本分类流程，降低了传统关键词法在跨地区文本比较中的误判率。

### 6.3 实证贡献

通过新的数据、案例、材料或观察发现增加经验知识。

常见形式：

- 发现新的经验模式；
- 验证或挑战既有结论；
- 提供新场景证据；
- 揭示异质性；
- 发现边界条件。

弱表达：

> 本文提供了新的数据。

强表达：

> 本文基于 2015—2024 年的跨省面板数据发现，政策强度对创新产出的影响主要出现在高市场化地区，从而揭示了制度环境的调节作用。

### 6.4 解释贡献

通过揭示机制、过程或因果路径解释现象。

常见形式：

- 解释为什么 A 导致 B；
- 解释相同原因为何导致不同结果；
- 解释异常案例；
- 揭示中介机制或过程链条。

弱表达：

> 本文分析了影响因素。

强表达：

> 本文表明，组织资源并非直接提升政策执行效果，而是通过提高跨部门协调能力间接发挥作用。

### 6.5 系统或工程贡献

通过设计、实现并评估一个系统、平台、工具或框架产生贡献。

常见形式：

- 新系统架构；
- 新工具链；
- 性能改进；
- 可用性提升；
- 工程约束下的创新解决方案；
- 开源数据或代码资源。

质量要求：

- 不能只说“实现了系统”；
- 必须说明系统解决了什么研究或工程问题；
- 必须有评价标准；
- 必须与已有系统比较。

### 6.6 规范贡献

常见于伦理、法学、政治理论、公共政策等领域。

常见形式：

- 提出规范判断框架；
- 澄清价值冲突；
- 论证某政策或制度安排为何正当；
- 提出原则、标准或评估维度；
- 批判已有规范假设。

质量要求：

- 概念清晰；
- 价值前提透明；
- 论证过程严密；
- 处理反方观点；
- 不把个人立场当结论。

### 6.7 综合贡献

有些论文贡献在于整合碎片化研究。

常见形式：

- 综述并重构领域知识图谱；
- 提出分类框架；
- 总结争议；
- 建立未来研究议程；
- 连接多个研究传统。

质量要求：

- 不是简单总结；
- 必须有组织框架；
- 必须说明该综合如何改变理解。

## 7. 证据与论证标准

### 7.1 什么是强证据

强证据具有以下特点：

1. **相关性**  
   证据直接对应研究问题或主张。

2. **透明性**  
   读者知道证据来源、选择标准和处理方式。

3. **充分性**  
   证据数量、质量或逻辑强度足以支持主张。

4. **可检验性**  
   其他研究者可以理解、复核、质疑或复现其路径。

5. **一致性**  
   证据与方法、概念和结论之间一致。

6. **抗替代解释能力**  
   作者考虑并处理了其他可能解释。

7. **边界清晰**  
   证据支持的范围被准确限定。

### 7.2 什么是弱证据

弱证据通常表现为：

- 单个例子支撑普遍结论；
- 只选择支持性材料；
- 数据来源不清；
- 没有说明样本选择；
- 主张比证据更大；
- 用权威引用代替分析；
- 用相关性表达因果；
- 用读者直觉代替论证；
- 用个案经验替代系统分析；
- 用模型输出替代解释。

### 7.3 论证强度判断

Agent 可用以下表格判断论证是否可靠：

| 论证问题 | 诊断问题 | 修改方向 |
|---|---|---|
| 主张过大 | 证据是否只能支持较小结论？ | 限定范围 |
| 证据不足 | 是否缺少数据、案例、文本或证明？ | 补充证据 |
| 逻辑跳跃 | 中间推理是否缺失？ | 增加 warrant |
| 概念混乱 | 核心术语是否前后一致？ | 重新定义概念 |
| 替代解释未处理 | 是否还有其他原因？ | 增加反驳或稳健性分析 |
| 贡献不明 | 相比已有研究新在哪里？ | 回到文献定位 |
| 方法不匹配 | 方法能否回答问题？ | 调整问题或方法 |

### 7.4 学术语体中的谨慎表达

科研论文应避免绝对化和宣传式表达。常用限定方式包括：

| 过度表达 | 更合适表达 |
|---|---|
| 证明了 X 一定导致 Y | 结果支持 X 与 Y 之间存在关联 / 可能关系 |
| 彻底解决了该问题 | 为理解该问题提供了新的解释 |
| 首次发现 | 本文在特定语境下揭示 |
| 完全颠覆已有研究 | 本文修正或补充了已有解释 |
| 所有情况下都适用 | 在本文研究条件下适用 |
| 必然导致 | 可能导致 / 在特定条件下导致 |

Agent 润色时应优先做三件事：

```text
1. 把绝对化表达改为有证据范围的表达；
2. 把空泛价值判断改为具体贡献；
3. 把宣传语改为可检验的学术主张。
```

## 8. 文献综述写法

### 8.1 文献综述的真正功能

文献综述应承担四个功能：

1. **构建问题空间**  
   说明哪些问题已经被提出，哪些问题仍待回答。

2. **构建争议空间**  
   说明已有研究之间的分歧、张力或不足。

3. **构建概念空间**  
   明确论文使用哪些概念，这些概念来自哪里，如何被定义。

4. **构建贡献位置**  
   说明本文将在哪个位置推进讨论。

### 8.2 从罗列文献到组织文献

弱综述：

> A 研究了 X。B 研究了 Y。C 认为 Z。D 提出了 W。

强综述：

> 现有研究大致形成三种解释路径。第一类研究强调制度因素，认为 X 主要源于 A；第二类研究关注组织能力，指出 B 是关键中介；第三类研究则从行动者认知出发，强调 C 的作用。虽然这些研究分别解释了 X 的不同方面，但它们较少讨论 A、B、C 如何在具体情境中共同作用。因此，本文将……

### 8.3 文献段落的推荐结构

每个文献综述段落可采用：

```text
主题句：指出本段讨论的研究方向。
综合句：概括这一方向的主要观点。
评价句：说明其贡献。
限制句：指出其不足或未解决问题。
连接句：说明与本文研究问题的关系。
```

### 8.4 缺口类型

常见缺口包括：

| 缺口类型 | 含义 | 风险 |
|---|---|---|
| 对象缺口 | 某对象研究较少 | 不能仅因少就值得研究 |
| 场景缺口 | 某语境下缺少研究 | 需说明该场景为何理论上重要 |
| 方法缺口 | 既有方法有限 | 需说明新方法能带来新知识 |
| 理论缺口 | 既有理论解释不足 | 需指出具体解释失败在哪里 |
| 机制缺口 | 知道相关性但不知道过程 | 需提供机制证据 |
| 争议缺口 | 既有研究结论不一致 | 需解释分歧来源 |
| 边界缺口 | 不知道结论何时成立 | 需研究条件或异质性 |
| 概念缺口 | 概念混用或未清晰界定 | 需提供概念澄清 |

### 8.5 Agent 判断规则

```text
文献综述不是越多越好，而是越能服务问题越好。
判断优先级：
1. 是否围绕研究问题？
2. 是否形成结构？
3. 是否综合而非罗列？
4. 是否指出具体缺口？
5. 是否自然引出本文贡献？
```

## 9. 结果、分析或论证主体的组织方式

### 9.1 主体部分的核心原则

主体部分必须服务于核心论点，而不是记录研究过程。组织方式应由以下因素决定：

- 研究问题数量；
- 贡献类型；
- 证据类型；
- 方法设计；
- 读者理解顺序；
- 论证递进关系。

### 9.2 通用段落结构

主体部分的分析段落可采用：

```text
小主张
→ 证据呈现
→ 证据解释
→ 与研究问题连接
→ 小结或过渡
```

示例骨架：

> 首先，A 表明 X 并非直接导致 Y，而是通过 B 发挥作用。表 2 显示……这一结果说明……结合前文理论框架，可以认为……因此，该发现回应了 RQ1 中关于机制路径的问题。

### 9.3 小节标题写法

弱标题：

```text
4.1 数据分析
4.2 结果展示
4.3 进一步讨论
```

强标题：

```text
4.1 规则透明度对公平性感知的直接影响有限
4.2 可申诉性强化了透明度的程序正义效应
4.3 高风险情境下透明度可能引发反向解释
```

好的小节标题应表达阶段性发现，而不仅是材料类型。

### 9.4 图表与正文的配合

正文不应逐项复述图表，而应回答：

```text
1. 图表中最重要的模式是什么？
2. 该模式如何回应研究问题？
3. 是否有异常或反直觉结果？
4. 该结果支持或挑战了哪个假设、命题或理论？
```

### 9.5 处理不一致结果

当结果与预期不一致时，不应回避。可采用：

```text
1. 明确指出结果与预期不同；
2. 检查方法或数据是否存在问题；
3. 提出可能解释；
4. 与已有研究比较；
5. 限定结论；
6. 将其转化为未来研究问题。
```

### 9.6 主体部分常见失败

| 失败模式 | 表现 | 修正方式 |
|---|---|---|
| 数据堆砌 | 大量表格但无解释 | 提炼每个结果的论点 |
| 材料堆砌 | 长段案例或访谈，无分析 | 增加编码、主题或机制解释 |
| 论点悬空 | 观点很多但无证据 | 为每个主张补证据 |
| 结构混乱 | 结果顺序随机 | 按研究问题或机制重组 |
| 过度解释 | 小结果推出大结论 | 限定主张范围 |
| 重复讨论 | Results 与 Discussion 无区别 | Results 讲发现，Discussion 讲意义 |

## 10. 讨论与结论写法

### 10.1 Discussion 与 Conclusion 的区别

| 部分 | 主要功能 | 重点 |
|---|---|---|
| Discussion | 解释结果的意义 | 为什么重要、与文献如何对话 |
| Conclusion | 压缩整篇论文 | 回答问题、总结贡献、指出边界 |

Discussion 更像“解释和对话”，Conclusion 更像“收束和定稿”。

### 10.2 讨论的四层提升

Agent 可帮助用户把结果逐层提升：

```text
结果层：我们发现了什么？
解释层：为什么会这样？
文献层：这与已有研究有什么关系？
贡献层：这改变了我们对什么问题的理解？
```

示例：

```text
结果层：透明度本身并不显著提升公平性感知。
解释层：用户不仅需要知道规则，还需要拥有质疑和申诉规则的渠道。
文献层：这修正了透明度研究中对信息公开作用的单向理解。
贡献层：本文提出“透明度—可申诉性耦合机制”，解释程序公平感知的形成。
```

### 10.3 贡献表达公式

```text
本文不是简单地证明 A，而是说明 A 只有在 B 条件下才会产生 C 结果。
```

```text
本文将 X 文献与 Y 文献连接起来，说明 Z 现象需要从二者交叉处理解。
```

```text
本文通过对 A 的分析，揭示了既有研究忽视的 B 机制。
```

```text
本文的发现扩展了 X 理论在 Y 情境中的适用边界。
```

### 10.4 局限性写法

局限性不应写成自我否定，而应写成边界说明。

弱写法：

> 本文样本较少，因此结论可能不准确。

较好写法：

> 本文基于特定类型的平台案例，适合解释高规则密度情境下的用户公平性感知。未来研究可进一步比较低规则密度平台，以检验本文机制的适用范围。

### 10.5 未来研究写法

未来研究应从本文边界自然推出，而不是泛泛而谈。

弱写法：

> 未来研究可以进一步扩大样本。

较好写法：

> 由于本文主要关注规则透明度与可申诉性的交互作用，未来研究可进一步考察不同用户群体如何基于经验差异解释同一治理规则。

## 11. 审稿人关注点

审稿人通常不是只看语言是否流畅，而是判断论文是否产生了可信且有价值的知识。常见关注点如下。

### 11.1 问题是否重要

审稿人会问：

```text
1. 这个问题为什么值得研究？
2. 它是学术问题，还是普通现象描述？
3. 它与领域核心争议有什么关系？
4. 读者为什么需要知道答案？
```

### 11.2 缺口是否真实

审稿人会问：

```text
1. 作者是否准确理解已有研究？
2. 所谓缺口是否只是作者没有读到相关文献？
3. 缺口是否具体？
4. 缺口是否重要到值得写一篇论文？
```

### 11.3 贡献是否清晰

审稿人会问：

```text
1. 本文到底新在哪里？
2. 新数据、新方法、新案例是否带来了新知识？
3. 贡献是理论、方法、实证、解释还是规范贡献？
4. 贡献是否被夸大？
```

### 11.4 方法是否匹配

审稿人会问：

```text
1. 方法能否回答研究问题？
2. 样本、案例、语料或数据是否合适？
3. 分析过程是否透明？
4. 是否存在选择偏差？
5. 是否考虑可靠性、有效性或可信度？
```

### 11.5 论证是否成立

审稿人会问：

```text
1. 主张是否由证据支持？
2. 是否存在逻辑跳跃？
3. 是否处理替代解释？
4. 是否区分相关、因果、解释和规范判断？
5. 是否承认边界？
```

### 11.6 结构是否服务论证

审稿人会问：

```text
1. 章节安排是否清晰？
2. 每章是否承担明确功能？
3. 文献、方法、结果、讨论是否彼此连接？
4. 是否存在重复、断裂或偏题？
```

### 11.7 语言是否符合学术语体

审稿人会问：

```text
1. 表达是否准确？
2. 是否过度声称？
3. 概念是否稳定？
4. 关键术语是否定义？
5. 是否有宣传式、情绪化或口号式语言？
```

## 12. 常见失败模式

### 12.1 主题大，问题小，贡献空

表现：

- 标题很宏大；
- 引言谈大背景；
- 研究问题模糊；
- 贡献只写“具有重要意义”。

修正：

```text
1. 将主题压缩为具体对象；
2. 将对象转化为可回答问题；
3. 将问题连接到文献缺口；
4. 将贡献写成对某类研究的具体推进。
```

### 12.2 文献综述变成文献列表

表现：

- 每段介绍一个作者；
- 没有综合；
- 没有争议；
- 没有自然导出本文问题。

修正：

```text
1. 按主题、争议、方法或理论流派重组；
2. 每组文献后加入评价；
3. 提炼已有研究共同假设；
4. 指出本文介入位置。
```

### 12.3 方法写得像流程记录

表现：

- 描述“我做了什么”；
- 不解释“为什么这样做”；
- 缺少选择标准和质量控制。

修正：

```text
1. 说明每个方法选择与研究问题的关系；
2. 交代数据、材料或案例选择理由；
3. 补充分析步骤；
4. 说明可信度保障方式。
```

### 12.4 结果没有论点

表现：

- 表格很多；
- 数据很多；
- 但读者不知道发现是什么。

修正：

```text
1. 为每个结果小节添加发现式标题；
2. 每段开头写小主张；
3. 图表之后解释意义；
4. 将结果连接回研究问题。
```

### 12.5 讨论重复结果

表现：

- Discussion 与 Results 内容相同；
- 没有理论意义；
- 没有与文献对话。

修正：

```text
1. 用一段简要回顾发现；
2. 随后解释为什么出现这些发现；
3. 与文献综述中的关键研究比较；
4. 提炼贡献和边界。
```

### 12.6 贡献表述空泛

表现：

- “丰富了相关研究”；
- “拓展了理论视野”；
- “具有重要实践意义”。

修正：

```text
把贡献改写为：
本文通过 X，说明 Y，从而修正/扩展/补充了 Z。
```

### 12.7 概念前后不一致

表现：

- 同一概念有多个说法；
- 关键术语未定义；
- 变量、维度和理论概念混用。

修正：

```text
1. 列出所有核心概念；
2. 给出工作定义；
3. 统一术语；
4. 检查每处使用是否一致。
```

### 12.8 结论超出证据

表现：

- 小样本推出普遍规律；
- 相关性写成因果；
- 个案写成普遍结论；
- 理论推演写成经验事实。

修正：

```text
1. 限定主张范围；
2. 使用谨慎表达；
3. 区分发现、解释、推测和建议；
4. 增加边界条件。
```

## 13. Agent 写作辅助策略

### 13.1 Agent 的一般工作流程

Agent 辅助科研论文写作时，应按以下顺序诊断，而不是直接润色句子：

```text
1. 识别论文类型
2. 提取研究主题
3. 判断研究问题
4. 定位文献缺口
5. 判断贡献类型
6. 检查方法/材料/理论路径
7. 检查证据和论证链
8. 检查章节功能
9. 重构结构
10. 润色语言
11. 检查标题、摘要、关键词
12. 输出修改建议或改写版本
```

### 13.2 第一步：识别论文类型

Agent 应先判断论文更接近哪种范式：

| 论文类型 | 主要证据形式 | 主体部分通常写法 |
|---|---|---|
| 量化实证 | 数据、模型、统计结果 | Hypotheses / Results |
| 实验研究 | 实验设计、实验结果 | Method / Results |
| 质性研究 | 访谈、观察、案例材料 | Findings / Analysis |
| 案例研究 | 案例过程、比较材料 | Case Analysis |
| 理论论文 | 概念、逻辑、理论对话 | Argument |
| 综述论文 | 文献集合、分类框架 | Review / Synthesis |
| 工程系统论文 | 系统设计、性能评价 | System / Evaluation |
| 计算论文 | 数据集、算法、实验 | Experiments / Results |
| 人文学科论文 | 文本、史料、解释 | Analysis / Interpretation |
| 规范论文 | 原则、价值、反驳 | Normative Argument |

判断规则：

```text
不要只看章节标题，要看论文依靠什么产生可信知识。
```

### 13.3 第二步：提取研究问题

Agent 可使用以下提问：

```text
1. 论文真正想回答的疑问是什么？
2. 这个问题是否能用一句话表达？
3. 它是描述性、解释性、因果性、机制性、规范性还是方法性问题？
4. 该问题是否能被当前材料回答？
5. 结论是否回到了这个问题？
```

输出格式：

```text
我识别出的研究问题是：
RQ1：……
可能的辅助问题：
RQ2：……
当前文本中的问题：
- 问题过宽：……
- 问题与方法不匹配：……
建议改写为：
……
```

### 13.4 第三步：判断贡献类型

Agent 应避免只写“理论意义和实践意义”。应判断具体贡献：

```text
本文的贡献更接近：
□ 理论贡献
□ 方法贡献
□ 实证贡献
□ 解释贡献
□ 系统贡献
□ 规范贡献
□ 综合贡献
```

然后生成贡献句：

```text
本文通过……，揭示/提出/修正/扩展/验证……，从而为……研究提供……
```

### 13.5 第四步：检查论证链

Agent 应把论文压缩为以下链条：

```text
已有研究认为 / 已经发现……
但是仍未解释 / 忽视 / 争议……
因此本文提出问题……
本文使用……
分析发现 / 论证……
这说明……
因此本文贡献是……
```

如果某一环缺失，应明确指出。

### 13.6 第五步：章节功能诊断

Agent 可按章节给出诊断表：

| 章节 | 当前问题 | 应承担功能 | 修改建议 |
|---|---|---|---|
| Introduction | 缺口不清 | 提出问题和贡献 | 增加文献缺口段 |
| Literature Review | 罗列文献 | 构建争议空间 | 按主题重组 |
| Method | 样本理由不足 | 说明证据生产 | 补充选择标准 |
| Results | 数据无解释 | 呈现发现 | 添加小结论 |
| Discussion | 重复结果 | 提炼意义 | 增加与文献对话 |
| Conclusion | 贡献空泛 | 收束全文 | 改写贡献句 |

### 13.7 第六步：重构而非只润色

当论文存在结构问题时，Agent 不应只改语言，而应提出重构方案。

常见重构动作：

- 合并重复段落；
- 将背景段压缩；
- 将文献综述按主题重排；
- 将结果小节改为发现式标题；
- 将 Discussion 从重复结果改为贡献讨论；
- 将结论改为“问题—发现—贡献—边界”结构；
- 将过度主张改为限定主张。

### 13.8 第七步：润色原则

Agent 润色科研论文时应优先保证：

```text
准确性 > 清晰性 > 连贯性 > 风格优美
```

不应为了语言漂亮而改变学术含义。

润色重点：

- 删除空泛表达；
- 增加逻辑连接；
- 统一概念术语；
- 减少绝对化表达；
- 强化主谓宾；
- 明确指代；
- 避免口语化；
- 保留必要限定。

## 14. 检查清单

### 14.1 全文总检清单

```markdown
## 全文逻辑
- [ ] 论文是否有明确研究问题？
- [ ] 研究问题是否具体、可回答？
- [ ] 引言是否说明问题重要性？
- [ ] 文献综述是否定位了真实缺口？
- [ ] 方法、材料或理论路径是否能回答问题？
- [ ] 结果、分析或论证是否围绕研究问题展开？
- [ ] Discussion 是否解释了结果意义？
- [ ] Conclusion 是否回到研究问题？
- [ ] 贡献是否具体？
- [ ] 边界是否清楚？

## 论证链
- [ ] 每个核心主张是否有证据？
- [ ] 证据是否与主张直接相关？
- [ ] 作者是否解释了证据为什么支持主张？
- [ ] 是否处理替代解释？
- [ ] 是否避免从相关性直接推出因果？
- [ ] 是否避免从个案直接推出普遍结论？
- [ ] 是否有过度声称？

## 文献使用
- [ ] 文献综述是否按主题、争议或理论组织？
- [ ] 是否避免逐篇罗列？
- [ ] 是否引用关键经典文献？
- [ ] 是否引用近期重要研究？
- [ ] 是否处理不同观点？
- [ ] 是否明确本文相对已有研究的新位置？

## 方法与证据
- [ ] 数据、材料、案例或语料来源是否清楚？
- [ ] 选择标准是否明确？
- [ ] 分析步骤是否透明？
- [ ] 质量控制是否说明？
- [ ] 方法限制是否承认？
- [ ] 伦理问题是否处理？

## 结构
- [ ] 每章是否承担不同功能？
- [ ] 小节标题是否反映论证内容？
- [ ] 章节之间是否有过渡？
- [ ] 是否存在重复段落？
- [ ] 是否存在与研究问题无关的内容？

## 语言
- [ ] 核心概念是否统一？
- [ ] 是否避免口号式表达？
- [ ] 是否使用谨慎限定？
- [ ] 是否避免模糊指代？
- [ ] 是否删除空泛句？
```

### 14.2 Introduction 检查清单

```markdown
- [ ] 是否在开头说明研究背景？
- [ ] 是否从背景收束到具体问题？
- [ ] 是否说明已有研究做了什么？
- [ ] 是否指出具体缺口？
- [ ] 是否明确提出研究问题？
- [ ] 是否说明本文研究路径？
- [ ] 是否预告主要发现或论点？
- [ ] 是否明确贡献？
- [ ] 是否提供文章结构安排？
```

### 14.3 Literature Review 检查清单

```markdown
- [ ] 是否围绕研究问题选择文献？
- [ ] 是否按主题、争议、方法或理论组织？
- [ ] 是否综合文献而非逐篇摘要？
- [ ] 是否评价已有研究不足？
- [ ] 是否界定核心概念？
- [ ] 是否指出本文介入位置？
- [ ] 是否避免只说“研究较少”？
```

### 14.4 Method / Data / Theory 检查清单

```markdown
- [ ] 是否说明研究设计？
- [ ] 是否说明材料、数据、案例或语料来源？
- [ ] 是否说明选择理由？
- [ ] 是否说明分析步骤？
- [ ] 是否定义变量、概念或编码标准？
- [ ] 是否说明可靠性、有效性或可信度？
- [ ] 是否说明伦理和限制？
```

### 14.5 Results / Analysis 检查清单

```markdown
- [ ] 是否按研究问题、假设、机制或主题组织？
- [ ] 每节是否有明确小结论？
- [ ] 是否提供充分证据？
- [ ] 是否解释证据意义？
- [ ] 是否处理异常结果？
- [ ] 图表是否被解释？
- [ ] 是否避免结果与讨论混淆？
```

### 14.6 Discussion / Conclusion 检查清单

```markdown
- [ ] 是否回扣研究问题？
- [ ] 是否解释核心发现？
- [ ] 是否与已有研究对话？
- [ ] 是否提炼贡献？
- [ ] 是否说明理论、方法、实践或规范意义？
- [ ] 是否说明边界？
- [ ] 是否提出合理未来研究？
- [ ] 结论是否避免引入新证据？
```
