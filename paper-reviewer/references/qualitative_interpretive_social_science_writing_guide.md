# 定性 / 解释型社会科学论文写作指南

## 1. 适用范围

本指南适用于以**访谈、田野观察、参与式研究、案例研究、档案材料、话语分析、文本解释、质性编码、叙事分析和情境化理解**为主要证据来源的定性 / 解释型社会科学论文。

适用领域包括但不限于：

- 社会学；
- 人类学；
- 教育学；
- 传播学；
- 组织研究；
- 管理学中的质性研究；
- 社会工作；
- 护理学；
- 公共卫生；
- 健康服务研究；
- 政策研究；
- 城市研究；
- 科技与社会研究；
- 性别研究；
- 文化研究；
- 法社会学；
- 政治社会学；
- 发展研究；
- 社区研究；
- 数字平台与社会生活研究。

适用研究材料包括：

- 半结构式访谈；
- 深度访谈；
- 焦点小组；
- 参与式观察；
- 非参与式观察；
- 田野笔记；
- 组织档案；
- 政策文本；
- 新闻报道；
- 社交媒体文本；
- 会议记录；
- 日记、书信、口述史；
- 影像、照片、海报等视觉材料；
- 案例材料；
- 实践记录；
- 行动研究材料；
- 护理、教育、社会工作或公共卫生现场资料；
- 质性编码数据；
- 混合方法研究中的质性部分。

这类论文的核心不是“讲很多故事”，而是：

> 通过有厚度的材料、透明的分析过程、深入的解释和敏锐的理论视角，说明某一社会现象、经验、过程、意义结构或制度机制如何被理解。

定性 / 解释型论文的可信度来自：

```text
材料厚度
+ 分析透明性
+ 解释深度
+ 理论敏感性
```

Agent 辅助这类论文时，不能只检查语言流畅，也不能把定性材料当作“轶事”。应重点判断：

```text
研究问题是否是解释性问题？
材料是否足以支撑解释？
主题是否从材料中建构出来？
分析过程是否透明？
作者是否处理情境、伦理和反思性？
结论是否从具体材料上升到理论贡献？
```

## 2. 不适用范围

本指南不应机械套用于以下论文或文本。

### 2.1 以变量关系和统计推断为核心的定量论文

如果论文主要通过问卷、实验、计量模型、大样本数据和统计识别来检验变量关系，应使用定量社会科学 / 行为科学论文指南。

定量论文通常问：

```text
X 是否影响 Y？
X 对 Y 的效应有多大？
Z 是否调节 X 与 Y 的关系？
```

定性论文通常问：

```text
行动者如何理解 X？
某种实践如何形成？
制度如何被经验化和协商？
某种意义、身份或关系如何在具体情境中被建构？
```

### 2.2 纯理论论文

如果论文主要依赖概念推演、规范论证或理论模型，而不使用经验材料，则不应套用本指南。

定性论文可以有强理论性，但必须让理论解释与具体材料发生关系。

### 2.3 普通新闻报道、调查纪实或案例介绍

新闻报道可以呈现故事和现场细节，但学术定性论文需要：

- 明确研究问题；
- 理论视角；
- 取样逻辑；
- 资料收集方法；
- 分析过程；
- 材料证据；
- 解释贡献；
- 伦理与反思性说明。

### 2.4 单纯访谈摘录汇编

只把受访者话语按主题排列，不解释话语背后的意义结构、社会机制、身份位置或制度情境，不构成强定性论文。

### 2.5 政策建议书或实践报告

政策建议书重在行动方案；定性论文重在解释现象、理解经验、揭示过程或建构理论。

### 2.6 只有“案例故事”但无分析框架的文本

案例研究不是讲故事。案例材料必须服务于某个解释性问题，并通过分析揭示超出个案本身的理论意义。

### 2.7 把定性研究写成“低配定量研究”

如果论文只是把访谈材料当作问卷结果的补充，用“多数人认为”“少数人表示”替代解释分析，通常没有发挥定性研究的优势。

## 3. 知识生产逻辑

定性 / 解释型社会科学论文通过以下方式生产可信知识：

```text
社会现象 / 经验困惑
→ 解释性研究问题
→ 理论视角或概念工具
→ 情境化资料收集
→ 深描与材料整理
→ 编码、比较、归纳或解释分析
→ 主题 / 概念 / 过程 / 类型建构
→ 材料证据支撑解释
→ 与已有理论对话
→ 形成情境化但可迁移的理论贡献
```

### 3.1 这类论文依靠什么产生可信知识

| 可信性来源 | 写作中应体现什么 | Agent 判断重点 |
|---|---|---|
| 材料厚度 | 访谈、观察、档案、文本是否足够丰富 | 是否有足够材料支撑解释 |
| 情境理解 | 对研究场域、行动者、制度背景的说明 | 是否脱离情境解释 |
| 分析透明性 | 取样、收集、编码、主题建构过程清楚 | 读者能否追踪解释来源 |
| 理论敏感性 | 能用概念工具看见材料中的结构和意义 | 是否只是材料摘要 |
| 解释深度 | 从“发生了什么”进入“为什么这样发生” | 是否有机制、过程或意义分析 |
| 反思性 | 研究者位置、关系和影响被说明 | 是否假装完全中立透明 |
| 伦理透明 | 同意、匿名、风险、关系处理清楚 | 是否保护参与者 |
| 负例和差异 | 不同声音、矛盾材料、反例被处理 | 是否只挑支持性材料 |
| 理论贡献 | 结果能修正、丰富或挑战已有理解 | 是否只是描述个案 |
| 可迁移性 | 说明结果适用于哪些相似情境 | 是否过度泛化 |

### 3.2 定性论文与定量论文的知识生产差异

| 维度 | 定性 / 解释型论文 | 定量社会科学论文 |
|---|---|---|
| 核心任务 | 理解意义、过程、经验和情境 | 检验变量关系、估计效应 |
| 研究问题 | 如何、为什么、以何种方式 | 是否、多大程度、因果效应 |
| 证据形式 | 访谈、观察、文本、案例、档案 | 样本、变量、模型、统计结果 |
| 推理方式 | 解释、归纳、比较、过程追踪、理论建构 | 假设检验、统计识别、模型估计 |
| 可信性来源 | 材料厚度、分析透明、反思性、理论敏感 | 测量有效、样本、识别、统计推断 |
| 结果形式 | 主题、机制、类型、叙事、概念 | 系数、效应量、显著性、模型结果 |
| 贡献方式 | 揭示意义结构、过程机制、概念修正 | 验证理论关系、估计效应、识别因果 |
| 常见风险 | 故事多但分析弱，理论大但材料薄 | 显著结果但理论弱，识别不足 |

定性研究不是为了用小样本“代表总体”，而是通过深入材料理解社会过程、意义建构和情境机制。它的目标通常不是统计泛化，而是：

```text
理论迁移
解释深化
机制揭示
概念生成
经验复杂化
```

### 3.3 定性论文的核心闭环

强定性论文应形成以下闭环：

```text
解释性问题
→ 理论视角
→ 情境和材料
→ 分析路径
→ 主题 / 机制 / 概念
→ 材料证据
→ 理论对话
→ 贡献和边界
```

常见断裂包括：

```text
研究问题问“如何形成”，但 Findings 只列观点。
理论部分谈制度理论，但材料分析只讲个人感受。
方法部分说扎根理论，但结果直接套用既有框架。
Findings 中有大量访谈摘录，但没有解释。
Discussion 中提出宏大理论主张，但材料只支持局部经验。
```

### 3.4 定性论文中的 Claim-Evidence-Warrant

定性论文同样需要“主张—证据—解释桥梁”。

| 元素 | 定性论文中的对应 |
|---|---|
| Claim | 主题、解释、机制、概念、过程判断 |
| Evidence | 访谈摘录、观察记录、文本材料、案例片段 |
| Warrant | 作者对材料的解释、比较、理论连接 |
| Boundary | 适用情境、材料限制、研究者位置 |

示例：

```text
Claim：平台劳动者并不只是被动接受算法控制，而是发展出“间歇性顺从”的实践策略。
Evidence：多名骑手描述他们会在高峰期严格遵守系统路线，但在低峰期主动绕路以维持熟悉片区。
Warrant：这种差异化行动说明劳动者会根据时间压力、订单密度和惩罚风险重新解释算法规则。
Boundary：该解释主要来自外卖骑手场景，不能直接推广到所有平台劳动。
```

Agent 检查规则：

```text
每当作者提出一个主题或解释时，必须检查：
1. 支撑材料在哪里？
2. 材料是否足够具体？
3. 作者是否解释了材料为什么支持该主题？
4. 是否有反例或不同声音？
5. 是否把个别叙述过度推广？
6. 是否回到理论问题？
```

## 4. 典型论文结构

定性 / 解释型社会科学论文常见结构如下：

```text
Title
Abstract
Keywords

1. Introduction
2. Literature Review / Theoretical Lens
3. Research Context
4. Methodology
5. Data / Materials
6. Findings / Themes / Analysis
7. Discussion
8. Conclusion

References
Appendices
```

不同传统会有变体：

- 人类学论文可能把 Research Context 和 Findings 融合在民族志叙事中；
- 社会学论文常有 Theory、Methods、Findings、Discussion；
- 组织研究常写 Theoretical Background、Methods、Findings、Discussion；
- 教育、护理、公共卫生质性研究常明确写 Sampling、Data Collection、Data Analysis、Trustworthiness；
- 话语分析论文可能突出 Corpus、Analytic Approach、Interpretive Findings；
- 案例研究论文可能突出 Case Selection、Case Narrative、Within-case / Cross-case Analysis；
- 扎根理论论文可能强调 Coding、Category Development、Theoretical Model；
- 行动研究或参与式研究会更重视参与关系、伦理和知识共创。

### 4.1 章节功能总览

| 章节 | 核心功能 | 不应写成 |
|---|---|---|
| Introduction | 提出解释性问题和研究意义 | 变量关系式问题 |
| Literature / Theoretical Lens | 建立理论视角和解释缺口 | 文献罗列 |
| Research Context | 说明场域、制度、文化和行动者背景 | 简单背景介绍 |
| Methodology | 说明研究立场、取样、资料收集和分析路径 | 技术流程清单 |
| Data / Materials | 说明材料来源、类型、范围和质量 | 材料堆砌 |
| Findings / Themes | 用材料支撑主题、机制和解释 | 访谈摘录合集 |
| Discussion | 将发现提升为理论贡献 | 重复 Findings |
| Conclusion | 回到研究问题、贡献和边界 | 空泛总结 |

### 4.2 结构质量判断

Agent 应检查：

```text
1. Introduction 是否提出解释性问题？
2. Literature 是否提供理论镜头，而非只综述主题？
3. Research Context 是否足以理解材料？
4. Methodology 是否透明说明资料如何产生和分析？
5. Findings 是否围绕主题或机制组织？
6. 材料是否支持解释？
7. Discussion 是否与理论对话？
8. Conclusion 是否说明贡献和边界？
```

## 5. 各章节写作指南

## 5.1 标题 Title

### 写作目标

标题应传达研究现象、场域、解释对象或核心概念，让读者知道论文关注什么社会经验、过程、意义或机制。

### 常见标题类型

| 类型 | 结构 | 示例形式 |
|---|---|---|
| 现象 + 场域 | X in Y Context | 某现象在某场域中的形成 |
| 概念型 | “X”: How Actors Make Sense of Y | 提炼核心概念 |
| 过程型 | Becoming X: The Process of Y | 经验或身份形成 |
| 矛盾型 | Between X and Y | 张力与协商 |
| 机制型 | How X is Negotiated in Y | 协商、建构、实践 |
| 主副标题 | 概念性主标题 + 解释性副标题 | “看不见的边界”：社区照护中的责任协商 |

### 常见错误

- 标题过于宽泛，如“农村教育研究”；
- 标题像定量论文，只写 X 与 Y；
- 标题过度文学化，读者看不出研究对象；
- 标题承诺宏大理论，但材料很局部；
- 标题不体现场域或人群；
- 标题只写方法，如“基于访谈的研究”。

### Agent 质量判断

```text
1. 标题是否说明研究现象？
2. 是否体现场域或对象？
3. 是否暗示解释重点或核心概念？
4. 是否避免过度抽象或过度文学化？
5. 是否与材料和贡献一致？
```

---

## 5.2 摘要 Abstract

### 写作目标

摘要应压缩完整定性研究逻辑：

```text
问题背景
→ 解释缺口
→ 材料和方法
→ 核心发现 / 主题
→ 理论贡献
→ 边界
```

### 推荐结构

```text
1. 背景句：指出研究现象或实践问题。
2. 缺口句：说明已有研究未充分解释什么。
3. 方法句：说明资料来源、场域、样本和分析方法。
4. 发现句：概括核心主题、过程或概念。
5. 贡献句：说明本文如何修正、丰富或挑战已有理论。
6. 边界句：说明适用情境和限制。
```

### 示例骨架

```text
平台劳动研究通常强调算法控制对劳动过程的约束，但较少解释劳动者如何在日常实践中理解、协商和调整这些规则。本文基于对 X 城市外卖骑手的 42 次深度访谈和三个月田野观察，分析平台劳动者与算法规则之间的互动过程。研究发现，骑手并非简单顺从或抵抗算法，而是形成了“间歇性顺从”“熟人化绕行”和“风险分层执行”三种实践策略。本文提出，算法控制的实际效力取决于劳动者对时间压力、惩罚风险和地方知识的情境化解释。该发现丰富了平台治理研究对劳动者能动性的理解，但结论主要适用于高流动性城市配送场景。
```

### 常见错误

- 摘要只写研究背景，不写发现；
- 不说明材料来源；
- 只说“采用访谈法”，不说明样本和场域；
- 发现写成“总结了若干问题”；
- 贡献写成“具有理论和实践意义”；
- 不说明研究边界；
- 把定性发现写成统计泛化。

### Agent 质量判断

```text
1. 摘要是否提出解释问题？
2. 是否说明材料和方法？
3. 是否概括核心主题或机制？
4. 是否说明理论贡献？
5. 是否避免空泛意义和过度泛化？
```

## 5.3 Introduction 引言

### 写作目标

定性论文的 Introduction 应提出**解释性问题**，而不是变量关系问题。

它应回答：

```text
有什么社会现象值得解释？
已有研究如何理解它？
还有什么经验、过程、意义或情境未被解释？
为什么需要定性材料？
本文将如何进入这个问题？
```

### 解释性问题示例

适合定性论文的问题：

```text
行动者如何理解某项政策？
某种身份如何在日常互动中形成？
组织成员如何在制度压力下协商实践？
患者如何解释疾病经验和医疗关系？
基层执行者如何将政策转化为具体行动？
某类话语如何构造问题和责任？
```

不适合作为定性主问题的写法：

```text
X 是否影响 Y？
X 对 Y 的影响程度有多大？
Z 是否调节 X 与 Y 的关系？
```

这些问题更适合定量研究。定性研究可以关注 X 与 Y 的关系，但通常应改写为：

```text
行动者如何理解 X 与 Y 的关系？
X 通过哪些实践过程与 Y 发生关联？
在具体情境中，X 如何被赋予意义并影响行动？
```

### 推荐组织方式

```text
1. 具体现象或经验困惑；
2. 现有研究如何解释；
3. 现有解释缺少什么；
4. 为什么需要深入材料或情境分析；
5. 本文研究问题；
6. 材料和方法简述；
7. 核心发现预告；
8. 理论贡献声明。
```

### 好的引言写法

弱写法：

```text
近年来，社区治理越来越重要，许多学者进行了研究。
```

强写法：

```text
在社区治理实践中，基层工作人员常常同时面对行政考核、居民需求和资源不足的多重压力。已有研究多从制度设计或治理能力解释基层执行困境，但较少关注基层工作者如何在日常互动中解释政策要求，并将抽象任务转化为具体行动。这一过程决定了政策在基层如何被“做出来”。
```

### 研究问题写法

```text
本文关注：基层工作人员如何在行政要求和居民需求之间理解、协商和执行社区治理任务？
```

```text
本文试图解释：平台劳动者如何在算法规则下形成日常行动策略？
```

```text
本文的问题不是评估政策是否有效，而是理解政策执行者如何在具体场域中定义“有效执行”。
```

### 常见错误

- 引言写成宏大社会背景；
- 问题仍是定量变量关系；
- 不说明为什么需要定性方法；
- 只说“研究较少”，没有解释缺口；
- 没有提出明确研究问题；
- 研究问题与材料不匹配；
- 贡献只写“丰富研究”；
- 过早介绍访谈人数而没有问题动机。

### Agent 质量判断

```text
1. 引言是否从具体社会现象切入？
2. 是否提出解释性问题？
3. 是否说明已有研究解释不足？
4. 是否说明为什么需要定性材料？
5. 是否预告核心发现或概念？
6. 是否明确理论贡献？
```

## 5.4 Literature / Theoretical Lens 文献综述与理论视角

### 写作目标

定性论文的文献综述不是“已有研究说了什么”的列表，而是为材料解释提供理论镜头和概念工具。

它应回答：

```text
本文用什么理论视角看材料？
哪些概念帮助解释现象？
已有研究解释了什么？
哪些经验或意义层面仍未被理解？
本文如何在材料中推进这些理论讨论？
```

### Theoretical Lens 的作用

理论视角不是用来压制材料，而是帮助研究者看见材料中的结构、关系和意义。

常见理论视角包括：

- 制度理论；
- 实践理论；
- 符号互动论；
- 规训与治理理论；
- 情感社会学；
- 身份理论；
- 组织意义建构；
- 文化资本与社会资本；
- 性别理论；
- 后殖民理论；
- 话语理论；
- 社会建构论；
- 生命历程理论；
- 生态系统理论；
- 交叉性理论；
- 专业主义理论；
- 政策执行理论；
- 街头官僚理论；
- 照护伦理；
- 风险社会理论。

### 文献综述组织方式

#### 方式一：理论视角组织

```text
制度压力如何被理解
日常实践如何中介制度
本文从意义建构视角补充
```

#### 方式二：概念工具组织

```text
算法控制
劳动者能动性
地方知识
本文用“情境化顺从”连接三者
```

#### 方式三：解释缺口组织

```text
已有研究解释结构约束
已有研究解释个体策略
但较少解释结构与策略如何在日常互动中连接
```

#### 方式四：争议组织

```text
一类研究强调控制
一类研究强调抵抗
本文说明控制和抵抗之间存在实践性协商
```

### 好的文献综述段落

```text
平台劳动研究常将算法视为一种强化劳动控制的技术安排。这一视角揭示了平台通过评分、派单和惩罚机制塑造劳动过程。然而，如果仅从控制角度理解平台劳动，容易忽略劳动者在具体城市空间、时间压力和经验知识中对算法规则的重新解释。本文引入实践理论，关注劳动者如何在日常行动中把算法规则转化为可操作的实践策略。
```

### 常见错误

- 文献综述按作者罗列；
- 理论视角只是名词装饰；
- 文献综述与 Findings 无关；
- 理论框架过于宏大，材料支撑不足；
- 只引用支持性研究，不处理不同解释；
- 理论提前决定发现，材料没有空间；
- 文献缺口只是“研究少”；
- 不说明本文如何使用概念。

### Agent 质量判断

```text
1. Literature 是否建立解释视角？
2. 是否定义核心概念？
3. 是否说明已有解释不足？
4. 理论视角是否适合材料？
5. 是否避免文献罗列？
6. Findings 是否会回到这些概念？
```

## 5.5 Research Context 研究情境

### 写作目标

Research Context 说明材料发生的社会、制度、文化、组织或历史背景。定性研究中的情境不是背景装饰，而是解释的一部分。

它应回答：

```text
研究发生在哪里？
这个场域有什么制度、文化或关系特征？
行动者面对什么约束和资源？
为什么这个场域适合回答研究问题？
读者需要知道哪些背景才能理解材料？
```

### 应包含内容

- 场域位置；
- 组织或社区背景；
- 政策制度背景；
- 历史发展；
- 关键行动者；
- 权力关系；
- 资源结构；
- 文化意义；
- 日常实践；
- 案例选择理由；
- 研究者进入场域方式。

### 好的情境写法

弱写法：

```text
本研究在 A 市进行。A 市经济发展较快，社区治理较为复杂。
```

强写法：

```text
A 市近年来推行网格化社区治理，将居民服务、治安巡查、矛盾调解和政策宣传整合到同一基层工作体系中。社区工作人员因此同时承担行政上传、居民服务和风险排查等多重任务。这个制度背景使 A 市成为观察基层工作者如何在行政要求与居民需求之间协商行动的合适场域。
```

### 常见错误

- 情境太少，读者无法理解材料；
- 情境太多，像背景报告；
- 不说明案例选择理由；
- 不说明场域特殊性和普通性；
- 忽略权力关系；
- 忽略研究者进入场域的方式；
- 情境与研究问题无关。

### Agent 质量判断

```text
1. 情境是否帮助解释研究问题？
2. 是否说明场域选择理由？
3. 是否介绍关键行动者和制度关系？
4. 是否足以理解材料摘录？
5. 是否避免背景资料堆砌？
```

## 5.6 Methodology 方法论

### 写作目标

Methodology 不只是“用了访谈法”，而是说明研究者如何生产、选择、解释材料。

它应回答：

```text
研究采取什么方法论立场？
为什么适合这个问题？
材料如何产生？
样本如何选择？
资料如何分析？
研究者如何处理反思性和伦理关系？
分析如何保证可信？
```

### 应包含内容

1. 研究设计；
2. 方法论取向；
3. 研究者立场；
4. 取样逻辑；
5. 资料来源；
6. 资料收集过程；
7. 资料分析过程；
8. 编码或解释路径；
9. 可信度策略；
10. 伦理审批、知情同意和匿名化；
11. 研究者反思性；
12. 局限和边界。

### 方法论取向

常见取向包括：

| 取向 | 重点 |
|---|---|
| 扎根理论 | 从材料中生成概念和理论 |
| 民族志 | 长期场域参与和文化深描 |
| 案例研究 | 深入解释一个或多个案例 |
| 现象学 | 经验、感受和意义结构 |
| 叙事分析 | 个人或群体如何组织故事 |
| 话语分析 | 语言如何构造对象、身份和权力 |
| 主题分析 | 识别和解释材料中的主题模式 |
| 档案研究 | 历史材料和制度过程解释 |
| 行动研究 | 研究者与实践者共同介入和反思 |
| 参与式研究 | 研究对象参与问题定义和知识生产 |

### 取样逻辑

定性研究通常不是追求随机代表性，而是追求材料与问题的解释相关性。

常见取样方式：

- 目的性取样；
- 理论取样；
- 最大差异取样；
- 滚雪球取样；
- 典型案例取样；
- 极端案例取样；
- 关键知情人取样；
- 分层目的性取样；
- 便利取样，但需说明限制；
- 案例比较取样。

### 取样写法

弱写法：

```text
本文访谈了 30 名受访者。
```

强写法：

```text
本文采用目的性取样，选择具有不同工作年限、平台类型和配送区域的骑手，以捕捉算法规则在不同劳动情境中的经验差异。随后通过滚雪球方式补充访问具有申诉、封号和跨平台经验的受访者，以增强材料对平台控制边界的解释力。
```

### 资料收集过程

应说明：

- 访谈时间；
- 访谈地点或方式；
- 访谈长度；
- 访谈提纲；
- 是否录音；
- 转录方式；
- 观察时间；
- 观察场景；
- 田野笔记记录方式；
- 档案材料来源；
- 文本收集标准；
- 资料语言和翻译；
- 资料管理。

### 资料分析过程

应说明：

- 采用何种分析方法；
- 编码阶段；
- 主题如何形成；
- 是否多人编码；
- 分歧如何处理；
- 是否使用软件；
- 是否反复比较材料；
- 是否寻找负例；
- 是否进行成员核对；
- 是否写分析备忘录；
- 主题与理论如何连接。

### 编码方法写法

```text
资料分析分三步进行。第一，研究者进行开放编码，标记受访者关于规则理解、工作策略和风险判断的叙述。第二，通过轴心编码将初始代码整合为“规则可预测性”“惩罚风险”“地方经验”等主题。第三，研究者比较不同工作场景中的主题组合，形成“间歇性顺从”这一核心概念。
```

### 可信度策略

定性研究常用可信度策略包括：

- 三角验证；
- 负例分析；
- 成员核对；
- 厚描；
- 同行讨论；
- 审计轨迹；
- 反思日志；
- 多研究者编码；
- 理论饱和或信息充分性说明；
- 案例比较；
- 原始材料和主题之间的透明连接。

注意：

```text
“饱和”不应作为空洞宣称。应说明何种主题达到饱和，如何判断没有出现新主题。
```

### 常见错误

- 只写“采用访谈法”；
- 不说明取样逻辑；
- 不说明访谈过程；
- 不说明编码过程；
- 使用“扎根理论”但没有开放、轴心、选择性编码或理论取样逻辑；
- 说“达到饱和”但无说明；
- 不说明研究者身份和关系；
- 不说明伦理；
- 把软件当方法；
- 分析过程不可追踪。

### Agent 质量判断

```text
1. 方法论是否匹配研究问题？
2. 取样逻辑是否清楚？
3. 材料收集过程是否透明？
4. 分析过程是否可追踪？
5. 是否说明反思性和伦理？
6. 是否说明可信度策略？
7. 是否避免方法名词装饰？
```

## 5.7 Data / Materials 资料与材料

Data / Materials 部分说明论文的经验基础。

它应回答：

```text
材料是什么？
来自哪里？
规模如何？
覆盖哪些人、场景、文本或时间？
材料如何被记录和管理？
材料是否足以支撑研究问题？
```

### 应包含内容

- 访谈人数；
- 受访者特征；
- 访谈次数和时长；
- 观察地点和时间；
- 档案材料范围；
- 文本语料规模；
- 案例数量；
- 材料选择标准；
- 材料排除标准；
- 资料保存和匿名化；
- 材料语言和翻译；
- 材料限制。

### 资料表格

访谈研究可使用表格：

| 编号 | 性别 / 年龄 | 角色 | 经验年限 | 访谈次数 | 备注 |
|---|---|---|---|---|---|

案例研究可使用表格：

| 案例 | 场域 | 选择理由 | 材料类型 | 时间范围 |
|---|---|---|---|---|

文本研究可使用表格：

| 材料类型 | 来源 | 时间范围 | 数量 | 选择标准 |
|---|---|---|---|---|

### 常见错误

- 不报告受访者基本信息；
- 不说明材料选择标准；
- 材料规模和研究问题不匹配；
- 只说“访谈若干人”；
- 不说明观察或文本材料范围；
- 材料描述过多但不服务分析；
- 不说明匿名处理；
- 不说明材料限制。

### Agent 质量判断

```text
1. 资料来源是否清楚？
2. 材料规模是否足以支撑分析？
3. 样本特征是否与研究问题相关？
4. 材料选择是否有逻辑？
5. 是否说明资料边界和限制？
```

## 5.8 研究者反思性与伦理

### 写作目标

定性研究中，研究者不是透明工具。研究者的身份、位置、关系和进入场域方式会影响材料产生和解释。

反思性不是自我表白，而是说明：

```text
研究者与场域的关系如何影响资料获取？
研究者身份如何影响受访者表达？
研究者如何处理权力不对称和伦理风险？
研究者如何反思自己的解释立场？
```

### 反思性应包含

- 研究者身份；
- 与场域关系；
- 进入场域方式；
- 与参与者关系；
- 语言、阶层、性别、专业身份可能影响；
- 研究者预设；
- 反思日志或团队讨论；
- 解释过程中的自我审查；
- 如何处理不对称权力关系。

### 反思性写法

弱写法：

```text
研究者保持客观中立。
```

强写法：

```text
研究者此前没有平台配送经验，这使访谈初期更依赖受访者解释行业术语和日常规则。为减少外部研究者视角带来的误读，研究者在访谈后整理术语表，并在后续访谈中向不同受访者核对关键实践描述。
```

### 伦理应包含

- 伦理审批；
- 知情同意；
- 自愿参与；
- 退出权利；
- 匿名化；
- 敏感信息处理；
- 录音和转录授权；
- 数据存储；
- 潜在风险；
- 对弱势群体保护；
- 研究回馈；
- 参与式研究中的权力共享。

### 常见错误

- 不写伦理；
- 只写“已匿名”；
- 不说明敏感材料如何处理；
- 不说明研究者与参与者关系；
- 不处理权力不对称；
- 反思性写成形式化段落；
- 参与式研究不说明参与者如何参与知识生产。

### Agent 质量判断

```text
1. 是否说明研究者位置？
2. 是否说明研究者与场域关系？
3. 是否说明伦理审批或同意过程？
4. 是否处理匿名和风险？
5. 是否解释反思性如何影响分析？
```

## 5.9 Findings / Themes 发现与主题

### 写作目标

Findings 是定性论文的核心。它应通过材料支撑主题、概念、过程和解释，而不是把材料堆起来让读者自己理解。

Findings 应回答：

```text
材料揭示了什么主题？
这些主题如何彼此关联？
它们如何回答研究问题？
每个主题由哪些材料支持？
作者如何解释材料？
```

### 组织方式

#### 方式一：主题式

适合访谈和主题分析。

```text
主题一：……
主题二：……
主题三：……
主题之间的关系
```

#### 方式二：过程式

适合解释形成过程。

```text
阶段一：进入场域
阶段二：学习规则
阶段三：协商实践
阶段四：形成身份
```

#### 方式三：机制式

适合揭示行动逻辑。

```text
机制一：意义重构
机制二：关系协调
机制三：风险规避
```

#### 方式四：类型式

适合建构行动者类型。

```text
类型一：顺从型
类型二：协商型
类型三：退出型
```

#### 方式五：案例比较式

适合多案例研究。

```text
案例 A：……
案例 B：……
跨案例比较：……
```

#### 方式六：叙事式

适合生命史、患者经验或身份研究。

```text
断裂
寻找解释
重新定位
形成新身份
```

### Findings 段落结构

推荐结构：

```text
1. 主题句：提出本段解释主张。
2. 材料呈现：访谈摘录、田野笔记或文本片段。
3. 材料解释：说明该材料如何体现主题。
4. 比较或扩展：引入其他材料、差异或负例。
5. 小结：回到研究问题或理论概念。
```

### 访谈摘录使用原则

访谈摘录不是越多越好。每段摘录应承担功能：

- 展示典型经验；
- 支持一个主题；
- 呈现矛盾或张力；
- 展示行动者语言；
- 呈现意义建构；
- 引出新概念；
- 说明例外或边界。

弱写法：

```text
受访者 A 说：“……”
受访者 B 说：“……”
受访者 C 说：“……”
因此可见大家都很重视社区参与。
```

强写法：

```text
受访者并未把“参与”理解为抽象的公共责任，而是将其与邻里关系中的互惠义务联系起来。居民 L 的叙述显示了这种理解：

> “……”

这段叙述中的关键并非参与本身，而是“以后有事也有人帮”的互惠预期。类似表述在多位居民访谈中反复出现，说明社区参与在该场域中被嵌入日常关系交换，而不是被理解为制度倡导的公民义务。
```

### 材料与解释的比例

Findings 中应避免两种极端：

```text
材料太多，作者分析太少。
理论太多，材料证据太少。
```

较好的比例是：

```text
解释主张
→ 足够材料
→ 作者解释
→ 比较 / 反例
→ 理论连接
```

### 处理负例和差异

强定性论文应处理：

- 不同受访者之间的差异；
- 与主题不一致的材料；
- 例外案例；
- 沉默和缺席；
- 矛盾叙述；
- 场域中的权力差异；
- 时间变化。

写法：

```text
并非所有受访者都以这种方式理解平台规则。新入职骑手更倾向于将系统指令视为不可协商的命令，而有经验骑手则更常将其视为可以根据路况重新解释的建议。这一差异说明，算法规则的实际效力与劳动者积累的地方知识有关。
```

### 常见错误

- Findings 只是访谈摘录堆砌；
- 每个主题都很宽泛，如“压力”“支持”“挑战”；
- 主题之间没有关系；
- 不说明主题如何形成；
- 只使用支持性材料；
- 不解释摘录；
- 材料不能支撑理论主张；
- 结果写成定量统计，如“多数人认为”；
- Findings 和 Discussion 分不清。

### Agent 质量判断

```text
1. 每个主题是否回答研究问题？
2. 主题是否具体、有解释力？
3. 是否有材料证据支持？
4. 材料后是否有作者解释？
5. 是否处理差异和负例？
6. 主题之间是否形成结构？
7. 是否避免材料堆砌？
```

---

## 5.10 Discussion 讨论

### 写作目标

Discussion 将具体材料中的发现提升为理论贡献。它应回答：

```text
这些发现改变了我们对什么现象的理解？
它们如何回应已有理论？
是否提出了新概念、新机制或新解释？
结果适用于哪些情境？
对实践或政策有什么启示？
```

### 推荐结构

```text
1. 概括核心发现；
2. 回到研究问题；
3. 与理论视角对话；
4. 说明理论贡献；
5. 说明实践或政策意义；
6. 讨论边界和局限；
7. 提出未来研究方向。
```

### 理论贡献写法

弱写法：

```text
本研究丰富了平台劳动研究。
```

强写法：

```text
本文的发现表明，算法控制并不只是通过规则约束劳动者，而是在劳动者对时间压力、惩罚风险和地方知识的情境化解释中被重新组织。因此，本文将平台劳动研究中“控制—抵抗”的二分框架推进为“情境化协商”的过程视角。
```

### 从材料到理论的提升

材料层：

```text
骑手会根据高峰和低峰调整对系统路线的遵守程度。
```

主题层：

```text
这种做法体现了“间歇性顺从”。
```

机制层：

```text
顺从程度取决于劳动者对惩罚风险和时间收益的权衡。
```

理论层：

```text
这说明算法控制的效果不是固定的，而是在劳动者日常实践中被情境化执行。
```

### 实践意义写法

实践意义应来自分析，而不是空泛建议。

弱写法：

```text
有关部门应加强管理。
```

强写法：

```text
如果平台治理只关注规则透明度，而忽视劳动者在不同城市空间中的地方知识，就可能强化形式合规而削弱实际效率。治理设计应考虑劳动者对规则的情境化执行经验。
```

### 常见错误

- Discussion 重复 Findings；
- 只说“丰富理论”；
- 理论贡献过大，材料支撑不足；
- 不说明结果边界；
- 实践建议与材料无关；
- 不处理研究者反思性影响；
- 未来研究泛泛。

### Agent 质量判断

```text
1. Discussion 是否回到理论问题？
2. 是否说明发现如何修正或扩展理论？
3. 是否从材料中提炼概念或机制？
4. 是否说明边界和局限？
5. 是否避免宏大但无材料支撑的贡献？
```

---

## 5.11 Conclusion 结论

### 写作目标

Conclusion 应简洁收束研究问题、材料、发现、贡献和边界。

### 推荐结构

```text
本文关注……
基于……
研究发现……
这些发现表明……
本文贡献在于……
由于……
未来研究可……
```

### 示例

```text
本文关注基层工作人员如何在行政要求和居民需求之间协商社区治理实践。基于对 A 市三个社区的田野观察和 38 次访谈，本文发现基层工作者通过“任务翻译”“关系缓冲”和“风险排序”三种实践将抽象政策要求转化为可执行行动。本文的贡献在于揭示政策执行并非简单的上下级传导，而是在基层日常互动中被重新定义和排序的过程。由于材料主要来自城市社区场域，未来研究可比较农村和街道层级的执行实践。
```

### 常见错误

- 只重复摘要；
- 结论太宏大；
- 不说明材料边界；
- 不回到研究问题；
- 不概括核心概念；
- 引入新材料；
- 以政策口号结束。

### Agent 质量判断

```text
1. 是否回答研究问题？
2. 是否概括核心主题或概念？
3. 是否说明理论贡献？
4. 是否明确情境边界？
5. 是否避免新材料和过度推广？
```

## 6. 常见贡献类型

定性 / 解释型社会科学论文的贡献通常包括以下形式。

### 6.1 概念贡献

从材料中提炼新概念，或重新界定已有概念。

示例：

```text
本文提出“间歇性顺从”概念，用以描述平台劳动者在不同风险情境下对算法规则的差异化执行。
```

质量要求：

- 概念来自材料；
- 概念有清晰定义；
- 概念能解释多个材料片段；
- 概念与已有概念有区别；
- 概念具有一定理论迁移性。

### 6.2 过程贡献

揭示某种社会实践、身份、制度或关系如何形成和变化。

示例：

```text
本文揭示新教师职业身份形成经历了“制度适应—角色冲突—实践重组”三个阶段。
```

质量要求：

- 有时间或阶段逻辑；
- 材料支持过程变化；
- 不只是主题并列。

### 6.3 机制贡献

解释某种现象为什么会发生，或行动者如何在情境中产生某种行动。

示例：

```text
本文说明基层政策执行中的形式主义并非单纯来自官僚惰性，而是源于考核压力、责任规避和居民关系维护之间的实践性协调。
```

质量要求：

- 机制有材料支持；
- 说明行动者逻辑；
- 连接结构条件和行动过程；
- 处理替代解释。

### 6.4 类型学贡献

构建行动者、实践、组织或经验类型。

示例：

```text
本文识别出患者面对慢性病管理时的三种应对类型：医学依从型、经验调适型和关系协商型。
```

质量要求：

- 分类标准清楚；
- 类型之间有差异；
- 类型来自材料比较；
- 类型有解释用途。

### 6.5 理论修正贡献

用材料挑战、补充或修正已有理论。

示例：

```text
本文发现街头官僚并非只在资源不足下自由裁量，也会在数字化系统约束下发展新的“技术性裁量”。
```

质量要求：

- 明确被修正的理论；
- 材料说明现有理论解释不足；
- 提出更细致解释。

### 6.6 情境化贡献

说明某理论在特定文化、制度、组织或历史情境中如何表现不同。

示例：

```text
本文展示专业主义在基层护理场域中并非表现为职业自治，而是表现为对患者家庭关系的情感协调能力。
```

质量要求：

- 情境特征清楚；
- 不是简单“某地案例”；
- 说明情境如何改变理论关系。

### 6.7 方法或材料贡献

通过新的材料来源、参与式方法、长期田野或跨材料比较产生贡献。

质量要求：

- 材料能看到以往研究看不到的问题；
- 方法不是装饰；
- 说明材料如何改变解释。

### 6.8 批判贡献

揭示被自然化、制度化或话语化的权力关系。

示例：

```text
本文通过话语分析说明“家庭责任”如何被政策文本用来将照护负担重新分配给女性家庭成员。
```

质量要求：

- 证据来自文本和话语结构；
- 不只是立场宣示；
- 分析语言、分类、沉默和权力。

## 7. 证据与论证标准

## 7.1 强定性证据的特征

```text
1. 材料与研究问题高度相关；
2. 资料来源多样或深度足够；
3. 取样逻辑清楚；
4. 分析过程透明；
5. 主题具体且有解释力；
6. 材料摘录和作者解释紧密连接；
7. 处理差异、矛盾和负例；
8. 理论概念与材料互相支撑；
9. 研究者反思性和伦理透明；
10. 结论限定在情境范围内。
```

### 强证据组合示例

| 主张 | 强证据组合 |
|---|---|
| 某实践存在 | 多位受访者叙述 + 观察记录 |
| 某机制成立 | 过程材料 + 不同角色比较 + 反例处理 |
| 某概念有解释力 | 多场景材料 + 清晰定义 + 与旧概念区分 |
| 某话语构造责任 | 文本模式 + 关键词 + 沉默和分类分析 |
| 某案例具有理论意义 | 案例选择理由 + 深描 + 与理论问题连接 |

## 7.2 弱定性证据的特征

弱证据常表现为：

- 只用一两句访谈支撑大结论；
- 只有材料，没有分析；
- 只有理论，没有材料；
- 主题过宽；
- 不说明取样；
- 不说明分析过程；
- 不处理反例；
- 只挑选最符合作者观点的材料；
- 将个案经验推广到所有群体；
- 把受访者说法直接当作事实；
- 不说明研究者位置；
- 不说明伦理处理。

## 7.3 材料摘录使用标准

好的摘录应满足：

```text
1. 与主题直接相关；
2. 具有足够情境信息；
3. 长度适中；
4. 摘录后有解释；
5. 不暴露敏感身份；
6. 能展示行动者语言或意义结构；
7. 不只是重复作者已经说过的话。
```

### 7.4 解释强度分级

| 材料情况 | 合适表达 | 不合适表达 |
|---|---|---|
| 多源材料反复出现 | 材料显示 / 表明 | 证明所有人都如此 |
| 少数案例但分析深入 | 该案例揭示 | 普遍存在 |
| 单一访谈摘录 | 该叙述提示 | 说明该群体 |
| 文本模式明确 | 话语中反复构造 | 社会现实就是 |
| 研究者推论 | 可以理解为 / 可能意味着 | 事实证明 |

## 8. 文献综述写法

### 8.1 定性文献综述的功能

定性论文文献综述应构建：

```text
理论视角
概念工具
解释缺口
研究位置
```

它不是为了穷尽所有研究，而是说明作者如何带着某种理论敏感性进入材料。

### 8.2 文献综述的好结构

```text
1. 现有研究如何解释该现象；
2. 这些解释看见了什么；
3. 它们忽略了什么经验、过程或意义；
4. 本文采用什么理论视角；
5. 该视角如何帮助重新理解材料；
6. 本文将在什么位置推进讨论。
```

### 8.3 从文献到理论视角

弱写法：

```text
许多研究关注平台劳动，也有研究关注算法管理。
```

强写法：

```text
平台劳动研究中的控制视角揭示了算法如何通过评分、派单和惩罚机制规训劳动过程。然而，这一视角往往将劳动者置于规则接受者的位置，较少解释劳动者如何在具体空间和时间压力中重新理解规则。本文因此引入实践理论，关注规则如何在日常行动中被情境化执行。
```

### 8.4 常见错误

- 罗列文献；
- 理论视角和材料无关；
- 文献缺口只是“研究少”；
- 没有概念工具；
- 没有解释为什么该理论视角适合；
- 理论过强，压过材料；
- 后文 Findings 不再使用理论概念。

### Agent 质量判断

```text
1. 文献综述是否导出解释问题？
2. 是否提供概念工具？
3. 是否说明已有解释不足？
4. 是否说明理论视角如何用于材料分析？
5. 是否避免罗列？
```

## 9. 结果、分析或论证主体的组织方式

### 9.1 Findings 的核心原则

Findings 应围绕“解释结构”组织，而不是围绕访谈问题、受访者类别或材料来源简单排列。

弱结构：

```text
5.1 受访者对政策的看法
5.2 受访者遇到的困难
5.3 受访者的建议
```

强结构：

```text
5.1 将政策翻译为可完成任务：基层执行的第一步
5.2 在居民关系中重新排序责任：执行过程中的情感劳动
5.3 以材料痕迹证明工作完成：考核压力下的可见化实践
```

### 9.2 主题命名

弱主题：

```text
压力
沟通
支持
问题
困难
```

强主题：

```text
责任下沉中的角色模糊
关系性信任的临时修复
可见化考核与隐性劳动
风险规避中的选择性执行
```

好的主题应：

- 具体；
- 有解释力；
- 能概括多段材料；
- 能回答研究问题；
- 与理论概念关联；
- 不是简单话题标签。

### 9.3 材料—分析—理论连接

每个主题应完成：

```text
主题主张
→ 材料证据
→ 作者解释
→ 与其他材料比较
→ 理论连接
```

### 9.4 多案例论文的组织

多案例研究可采用：

#### 方式一：先个案后比较

```text
案例 A 深描
案例 B 深描
案例 C 深描
跨案例主题比较
```

#### 方式二：按主题跨案例

```text
主题一：三个案例中的不同表现
主题二：三个案例中的不同表现
主题三：机制比较
```

选择原则：

```text
如果案例本身复杂，需要先个案叙事；
如果目标是理论比较，可按主题组织。
```

### 9.5 话语分析主体组织

话语分析应关注：

- 关键词；
- 分类方式；
- 主体位置；
- 隐喻；
- 叙事结构；
- 责任归因；
- 沉默；
- 对立建构；
- 合法化策略。

不应只总结文本内容。

### 9.6 档案 / 历史材料主体组织

可按：

- 时间阶段；
- 制度变迁；
- 关键事件；
- 话语变化；
- 行动者关系；
- 政策逻辑；
- 案例过程。

### 9.7 常见主体失败

| 失败模式 | 表现 | 修正 |
|---|---|---|
| 访谈问题式结构 | 按访谈提纲写结果 | 按主题和机制重组 |
| 摘录堆砌 | 长引文连续出现 | 每段加入解释 |
| 主题过宽 | 压力、挑战、支持 | 提炼更具体概念 |
| 理论悬空 | 大理论后无材料 | 降低理论范围或补材料 |
| 材料不足 | 一条引文支撑大主张 | 增加材料或限定 |
| 无负例 | 只选支持材料 | 处理差异和例外 |

## 10. 讨论与结论写法

### 10.1 Discussion 的四层提升

```text
材料层：受访者说了什么 / 场域发生了什么？
主题层：这些材料形成什么主题？
机制层：主题之间如何连接成解释？
理论层：这如何改变已有理论理解？
```

### 10.2 理论贡献表达公式

```text
本文不是简单描述……，而是说明……如何在……情境中被行动者解释和实践。
```

```text
本文将已有研究中的“结构约束”理解推进为“结构—实践—意义”的动态过程。
```

```text
通过提出……概念，本文揭示了既有理论未能解释的……经验。
```

```text
本文发现挑战了……理论中关于……的隐含假设。
```

### 10.3 实践意义表达

实践意义应来自材料解释。

```text
如果基层执行者将政策理解为“可被检查的材料生产”，那么提升政策执行质量不能仅靠增加考核指标，还需要改变考核如何定义“有效工作”。
```

### 10.4 边界写法

```text
本文材料来自城市社区治理场域，因此更适合解释高度行政化社区中的执行实践。对于自治传统较强或资源结构不同的社区，本文机制是否成立仍需进一步比较。
```

### 10.5 未来研究写法

未来研究应从边界自然推出：

```text
未来研究可比较不同制度压力强度下基层工作者的任务翻译实践，以检验本文提出的“风险排序”机制是否具有跨场域适用性。
```

## 11. 审稿人关注点

### 11.1 材料是否足够

审稿人会问：

```text
1. 材料数量和深度是否支撑问题？
2. 取样是否有逻辑？
3. 场域是否足够说明？
4. 是否只访问了容易接触的人？
5. 是否缺少关键行动者声音？
```

### 11.2 分析是否透明

审稿人会问：

```text
1. 资料如何收集？
2. 如何编码？
3. 主题如何形成？
4. 是否有分析备忘录或比较过程？
5. 是否能追踪从材料到主题的过程？
```

### 11.3 主题是否牵强

审稿人会问：

```text
1. 主题是否真的来自材料？
2. 材料是否足以支撑主题？
3. 主题之间是否有区别？
4. 是否只是作者套理论？
5. 是否处理反例？
```

### 11.4 反思性是否缺失

审稿人会问：

```text
1. 研究者如何进入场域？
2. 研究者身份是否影响材料？
3. 受访者是否可能迎合研究者？
4. 权力关系和伦理风险如何处理？
```

### 11.5 理论贡献是否明确

审稿人会问：

```text
1. 本文只是讲了一个案例，还是提出理论解释？
2. 核心概念是什么？
3. 发现如何修正已有理论？
4. 是否有可迁移性？
```

### 11.6 写作是否平衡

审稿人会问：

```text
1. 是否材料多但分析少？
2. 是否理论大但材料薄？
3. 是否摘录过长？
4. 是否过度概括？
5. 是否有清晰论证主线？
```

## 12. 常见失败模式

### 12.1 故事很多但分析不足

表现：

- 访谈摘录很多；
- 案例情节丰富；
- 但作者解释很少。

修正：

```text
1. 每段材料前加主题主张；
2. 每段材料后加解释；
3. 比较不同材料；
4. 提炼概念而非继续讲故事；
5. 回到研究问题。
```

### 12.2 理论很大但材料支撑不足

表现：

- 使用宏大理论；
- 材料只支持局部经验；
- Discussion 过度上升。

修正：

```text
1. 降低理论主张范围；
2. 将贡献写成情境化修正；
3. 增加材料证据；
4. 明确边界。
```

### 12.3 主题太宽泛

表现：

```text
挑战、压力、支持、认同、沟通。
```

修正：

```text
1. 问“这是什么类型的压力？”
2. 问“这种压力如何产生？”
3. 问“它导致什么行动？”
4. 将话题标签改为解释性主题。
```

### 12.4 方法不透明

表现：

- 不知道如何访谈；
- 不知道如何编码；
- 不知道主题如何形成。

修正：

```text
1. 补取样逻辑；
2. 补资料收集过程；
3. 补编码和分析步骤；
4. 说明可信度策略。
```

### 12.5 缺少研究者反思性

修正：

```text
1. 说明研究者身份和场域关系；
2. 说明进入场域方式；
3. 说明可能影响；
4. 说明如何减少误读。
```

### 12.6 访谈摘录被当作事实

表现：

```text
受访者说政策执行很好，因此政策执行很好。
```

修正：

```text
1. 将受访者话语视为意义表达；
2. 与观察或其他材料比较；
3. 分析话语产生的情境；
4. 区分叙述与事实。
```

### 12.7 不处理差异和反例

修正：

```text
1. 寻找不符合主题的材料；
2. 比较不同群体；
3. 说明主题边界；
4. 用反例修正概念。
```

### 12.8 结论过度泛化

表现：

- 单一场域材料推广到所有社会；
- 少数访谈推广到总体。

修正：

```text
1. 使用情境化表述；
2. 强调理论迁移而非统计泛化；
3. 说明适用边界；
4. 提出未来比较研究。
```

## 13. Agent 写作辅助策略

### 13.1 Agent 总体流程

Agent 辅助定性论文时，应按以下流程工作：

```text
1. 识别研究问题是否解释性；
2. 判断理论视角是否适合；
3. 检查研究情境是否清楚；
4. 检查材料来源和取样逻辑；
5. 检查资料收集过程；
6. 检查分析和编码透明性；
7. 从材料中提炼主题；
8. 检查主题是否被材料支持；
9. 处理差异、负例和边界；
10. 将 Findings 上升为理论贡献；
11. 降低过度泛化；
12. 改写成清晰论证结构。
```

### 13.2 Agent 如何从原始材料中提炼主题

建议步骤：

```text
1. 逐段标注材料中的行动、情感、关系、分类和隐喻。
2. 将相似代码合并为初始主题。
3. 比较不同受访者、场景或文本中的差异。
4. 寻找主题背后的过程或机制。
5. 给主题命名，避免宽泛词。
6. 为每个主题寻找代表材料和负例。
7. 将主题排序为解释结构。
```

### 13.3 材料到主题的转换模板

原始材料：

```text
受访者反复提到“怕被投诉”“怕系统扣分”“熟路比系统路线快”。
```

初始代码：

```text
惩罚风险、路线经验、系统不信任、时间压力。
```

主题：

```text
在惩罚风险和地方知识之间进行路线判断。
```

概念：

```text
风险分层执行。
```

理论解释：

```text
算法规则的执行效果取决于劳动者对风险和地方知识的实践性权衡。
```

### 13.4 Agent 如何检查解释是否被材料充分支持

建立“主题—材料—解释”矩阵：

| 主题 / 概念 | 支持材料 | 材料类型 | 作者解释 | 是否充分 | 缺口 |
|---|---|---|---|---|---|
| 间歇性顺从 | 访谈 A、B、观察记录 | 访谈 + 田野 | 风险情境影响顺从 | 较充分 | 缺少新手视角 |
| 风险排序 | 访谈 C、政策文本 | 访谈 + 文本 | 考核压力重排任务 | 部分充分 | 需更多现场材料 |

Agent 判断：

```text
1. 每个主题至少有多条材料支撑吗？
2. 是否有不同来源材料互证？
3. 材料是否只是例子，还是能展示机制？
4. 作者是否解释了材料？
5. 是否处理反例？
6. 结论是否超出材料？
```

### 13.5 Agent 如何改写 Findings

弱段落：

```text
很多受访者表示工作压力大。A 说……B 说……C 说……
```

改写方向：

```text
本节应先提出解释性主题：
“压力并非单纯来自工作量，而是来自多重责任标准之间的不一致。”

然后选择 1—2 段最能体现这种张力的材料，解释其语言和情境，再比较不同受访者如何应对这种张力。
```

### 13.6 Agent 如何检查理论贡献

问题清单：

```text
1. 本文提出了什么新概念？
2. 是否揭示了新过程？
3. 是否修正了已有理论？
4. 是否说明已有理论在某情境下不足？
5. 是否解释了一个经验悖论？
6. 贡献是否有材料支撑？
```

### 13.7 Agent 如何降级过度声称

高风险表达：

```text
所有
普遍
证明
揭示了整个机制
完全说明
必然
该群体都
政策失败的根本原因
```

改写：

| 过强表达 | 更稳妥表达 |
|---|---|
| 所有基层干部都如此 | 在本文研究的社区中，多位基层工作人员表现出类似实践 |
| 本文证明政策失败 | 本文揭示了该政策在基层执行中面临的一种实践性困境 |
| 这说明平台劳动的本质 | 这为理解平台劳动中的算法规则执行提供了一个情境化视角 |
| 该机制普遍存在 | 该机制可能适用于具有类似考核压力和资源约束的场域 |
