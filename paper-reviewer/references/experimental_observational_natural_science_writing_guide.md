# 实验 / 观察型自然科学论文写作指南

## 1. 适用范围

本指南适用于以**实验、观测、测量、表征和自然现象解释**为核心的自然科学论文，尤其适合物理、化学、生物、材料、环境、地球科学、农学等领域。

这类论文通常通过以下方式生产知识：

- 在受控条件下改变变量，观察系统响应；
- 在自然条件下进行观测、采样或长期监测；
- 使用仪器、表征技术或测量方法获取数据；
- 通过对照、重复、误差估计和统计检验判断结果是否可信；
- 通过多组证据解释自然现象、材料性质、生物过程、环境变化或地球系统机制；
- 通过实验链条或观测链条支持一个明确科学问题的回答。

本指南适合辅助以下论文类型：

- 实验物理论文；
- 化学合成与反应机理论文；
- 生物实验论文；
- 材料制备、表征与性能论文；
- 环境监测与污染过程论文；
- 地球科学观测与解释论文；
- 农学田间试验论文；
- 生态观测或控制实验论文；
- 仪器测量与自然现象验证论文；
- 多方法表征与机制解释论文。

这类论文的核心不是“做了很多实验”，而是：

> 用一组有逻辑、有对照、有重复、有误差意识的证据链，回答一个明确的科学问题。

---

## 2. 不适用范围

本指南不应机械套用于以下论文类型。

### 2.1 纯理论或数学推导论文

如果论文主要依靠理论模型、数学证明或形式推导，而不是实验或观测数据生产知识，则不应套用本指南。  
这类论文的重点是模型假设、推导严密性、定理证明或理论解释力。

### 2.2 纯计算模拟论文

如果论文主要依靠数值模拟、分子动力学、气候模型、第一性原理计算或机器学习预测，而没有实测、实验或观测数据作为核心证据，应使用计算 / 模拟类论文写作范式。

但如果计算只是辅助实验解释，例如用 DFT 支持催化机制、用模型解释实验曲线，则本指南仍可作为主体框架。

### 2.3 临床试验或医学干预研究

临床研究涉及特定伦理、注册、随机化、盲法、受试者招募和安全性报告要求，不应简单套用一般自然科学实验论文范式。

### 2.4 工程系统实现论文

如果论文主要贡献是系统架构、算法实现、设备工程、软件平台或性能优化，而不是自然现象解释或实验测量，应使用工程系统论文指南。

### 2.5 综述论文

综述论文通过整合已有文献生产知识，不直接通过新实验或观测生产证据。  
它的核心是分类、综合、批判和研究议程，而不是实验设计与数据解释。

### 2.6 单纯检测报告或实验记录

如果文本只是记录“做了哪些实验、测了哪些指标、得到了哪些数值”，而没有研究问题、对照逻辑、证据链和科学解释，则不构成完整科研论文。

### 2.7 产品测试、质检报告或工程验收报告

这类文本强调是否达标、是否合格、性能是否满足规范。  
科研论文强调的是：结果如何回答科学问题、解释机制或推进知识。

## 3. 知识生产逻辑

实验 / 观察型自然科学论文的知识生产逻辑可以概括为：

```text
自然现象 / 科学问题
→ 已有解释不足
→ 提出假设或研究问题
→ 设计实验 / 观测 / 测量方案
→ 设置变量、对照、重复和误差控制
→ 获取数据和表征结果
→ 识别模式、差异、趋势或机制线索
→ 用多组证据建立解释链条
→ 限定结论适用条件
→ 形成科学贡献
```

### 3.1 这类论文依靠什么产生可信知识

可信知识不是来自“实验数量多”，而是来自以下因素的组合。

| 可信性来源 | 写作中应体现什么 | Agent 判断重点 |
|---|---|---|
| 明确科学问题 | 研究要解释什么现象、性质、机制或关系 | 问题是否具体 |
| 可辩护的实验 / 观测设计 | 为什么这样设计实验、采样、测量或比较 | 设计是否能回答问题 |
| 变量控制 | 自变量、因变量、控制变量、干扰因素 | 是否排除关键混杂 |
| 对照设置 | 空白对照、阴性对照、阳性对照、基准样品、对照组 | 对照是否能支持解释 |
| 重复与再现 | 技术重复、生物重复、独立重复、批次重复 | 结果是否稳定 |
| 测量可靠性 | 仪器校准、检测限、误差范围、数据处理 | 测量是否可信 |
| 统计与不确定性 | 误差条、置信区间、显著性、效应量 | 是否量化不确定性 |
| 多证据收敛 | 不同实验、表征或观测共同支持同一解释 | 是否形成证据链 |
| 机制解释 | 数据如何说明过程、机制或因果关系 | 是否只是描述 |
| 边界限定 | 在什么条件下成立，不能推广到哪里 | 是否过度声称 |

### 3.2 实验型与观察型论文的区别

#### 实验型论文

实验型论文通常通过主动操控变量来回答问题。

典型逻辑：

```text
改变 X
→ 控制其他条件
→ 测量 Y
→ 与对照比较
→ 判断 X 是否影响 Y
→ 解释影响机制
```

适合问题：

- 某因素是否影响某性质？
- 某处理是否改变某过程？
- 某材料结构为何提升性能？
- 某反应条件如何影响产物？
- 某基因、蛋白、化学物质或环境因子是否发挥作用？

写作重点：

- 自变量和因变量必须清楚；
- 对照组必须合理；
- 实验条件必须可重复；
- 结果必须包含误差和重复；
- 机制解释必须由实验链条支持。

#### 观察型论文

观察型论文通常不能直接操控自然系统，而是通过采样、监测、遥感、现场测量或长期数据分析识别模式。

典型逻辑：

```text
选择自然系统 / 样点 / 时间序列
→ 说明采样设计
→ 测量关键变量
→ 识别空间、时间或过程模式
→ 排除或讨论混杂因素
→ 提出解释或机制假说
```

适合问题：

- 某自然现象如何随时间变化？
- 某环境变量如何与生态过程相关？
- 某地质、气候或水文过程表现出什么模式？
- 某区域、样带或深度梯度中存在何种变化规律？
- 某自然事件的影响范围、强度和机制是什么？

写作重点：

- 样点、时间、空间尺度必须清楚；
- 采样策略必须有理由；
- 代表性和偏差必须讨论；
- 相关关系不能轻易写成因果；
- 机制解释应谨慎，最好有多源证据支持。

### 3.3 从“实验很多”到“证据链清楚”

弱论文常见逻辑：

```text
我们做了实验 A、B、C、D、E。
每个实验都有结果。
因此本文很完整。
```

强论文应形成逻辑：

```text
实验 A 证明样品 / 现象存在。
实验 B 证明关键变量发生变化。
实验 C 证明这种变化影响目标性质。
实验 D 排除替代解释。
实验 E 支持机制解释。
因此，本文回答了科学问题 X。
```

Agent 辅助写作时，应把实验列表重构为“发现链条”：

| 实验 / 观测 | 功能 | 支持的中间结论 |
|---|---|---|
| 基础表征 | 确认研究对象或样品性质 | 系统确实具备目标特征 |
| 对照实验 | 证明变化不是偶然或背景效应 | 处理因素具有特定作用 |
| 参数实验 | 说明变量与结果的关系 | 发现趋势或最优条件 |
| 机制实验 | 验证过程、路径或中间体 | 支持机制解释 |
| 稳健性实验 | 检验重复性、泛化性或边界 | 结论在特定范围内可靠 |

### 3.4 自然科学论文中的 Claim-Evidence-Warrant

自然科学论文中的每个重要结论都应符合：

```text
Claim：本文主张什么？
Evidence：哪个实验、观测、图表或统计结果支持它？
Warrant：为什么这个结果能支持该主张？
Limitation：这个主张在哪些条件下成立？
```

示例：

```text
Claim：处理 A 提高了材料的催化活性。
Evidence：在相同测试条件下，A 处理样品的电流密度高于对照组，并在多次重复中保持稳定。
Warrant：由于样品负载量、测试电解质和电极面积已保持一致，性能差异可归因于处理 A 引起的表面结构变化。
Limitation：该结论目前只适用于本文测试的电解质浓度和电位范围。
```

Agent 检查规则：

```text
每当作者写“表明”“证明”“揭示”“由于”“机制是”时，必须检查：
1. 这个判断由哪张图或哪组数据支持？
2. 是否有必要对照？
3. 是否有重复和误差？
4. 是否排除了明显替代解释？
5. 是否把相关性误写成因果？
6. 是否超出实验条件？
```

## 4. 典型论文结构

实验 / 观察型自然科学论文通常采用 IMRaD 结构：

```text
Title
Abstract
Keywords

1. Introduction
2. Materials and Methods / Methods / Experimental Section
3. Results
4. Discussion
5. Conclusion

Acknowledgements
Author Contributions
Data Availability
References
Supplementary Information / Appendices
```

部分期刊或学科会将 Results 与 Discussion 合并为 “Results and Discussion”。材料、化学和生物领域也常出现 “Experimental Section”“Materials and Characterization”“Study Area and Sampling”“Analytical Methods”等变体。

### 4.1 IMRaD 在自然科学论文中的功能

| 部分 | 核心问题 | 功能 |
|---|---|---|
| Introduction | 为什么研究？ | 从现象、机制问题和文献不足引出研究问题 |
| Methods | 如何研究？ | 说明实验、观测、测量和分析方法，使结果可评价、可重复 |
| Results | 发现了什么？ | 呈现数据、图表、表征和主要结果 |
| Discussion | 这些结果意味着什么？ | 解释机制、比较文献、处理异常、限定结论 |
| Conclusion | 最终回答是什么？ | 压缩核心发现、贡献和边界 |

### 4.2 常见结构变体

#### 变体一：Results and Discussion 合并

适合实验链条紧密、每个结果都需要立即解释的论文。

```text
1. Introduction
2. Methods
3. Results and Discussion
4. Conclusion
```

风险：

- 容易把结果和推测混在一起；
- 容易在每个小节中重复背景；
- 需要小节标题清晰表达发现链条。

#### 变体二：材料 / 样品先行

常见于材料、化学、农学、生物实验论文。

```text
1. Introduction
2. Materials and Methods
3. Sample Preparation / Characterization
4. Performance / Response / Functional Results
5. Mechanism Analysis
6. Discussion
7. Conclusion
```

#### 变体三：观察型现场研究

常见于环境、生态、地球科学、农学田间研究。

```text
1. Introduction
2. Study Area
3. Sampling and Methods
4. Results
5. Discussion
6. Conclusion
```

#### 变体四：多尺度证据链结构

适合从宏观现象到微观机制的论文。

```text
1. Introduction
2. Methods
3. Macroscopic Observation
4. Microscopic / Molecular Evidence
5. Mechanistic Interpretation
6. Discussion
7. Conclusion
```

### 4.3 结构质量判断

Agent 应检查：

```text
1. Introduction 是否提出明确科学问题？
2. Methods 是否足以让同行判断实验可靠性？
3. Results 是否按照证据链组织，而不是按实验时间顺序罗列？
4. Discussion 是否解释机制，而不是重复图表？
5. Conclusion 是否回答问题，而不是泛泛总结“性能提高”？
6. 图表顺序是否构成逻辑递进？
7. 结论是否被数据支持并限定在实验范围内？
```

## 5. 各章节写作指南

## 5.1 标题 Title

### 写作目标

标题应准确传达研究对象、关键变量、方法或主要发现，使同行快速判断论文的科学问题和贡献。

### 应包含内容

自然科学论文标题常包含：

- 研究对象：材料、体系、生物体、区域、过程、反应、样品；
- 关键变量：温度、浓度、处理、结构、环境因子、基因、成分；
- 主要响应：性能、机制、转化效率、稳定性、通量、表达水平；
- 方法或尺度：原位观测、长期监测、多尺度表征、田间试验；
- 核心发现：增强、抑制、调控、揭示、驱动、限制。

### 常见标题类型

| 类型 | 示例结构 | 适用场景 |
|---|---|---|
| 对象 + 发现 | X enhances Y through Z | 机制或性能发现 |
| 变量 + 响应 | Effects of X on Y under Z conditions | 控制实验 |
| 方法 + 对象 | In situ observation of X during Y | 表征或观测 |
| 机制型 | Z-mediated regulation of X in Y | 生物、化学、材料机制 |
| 场景型 | Seasonal variation of X in Y region | 环境、地球、生态观测 |
| 主副标题 | X under stress: evidence from Y | 需要兼顾可读性和精确性 |

### 常见错误

- 标题只写研究对象，不写问题；
- 标题过度声称机制，但正文证据不足；
- 标题使用 “novel”“excellent”“remarkable” 等宣传词；
- 标题承诺普遍规律，但实验条件很窄；
- 标题过长，堆叠过多技术和变量；
- 标题暗示因果，但研究只是相关观测。

### Agent 质量判断

```text
1. 标题是否体现研究对象？
2. 是否体现科学问题或核心发现？
3. 是否避免夸大机制或普适性？
4. 是否与图表和结论一致？
5. 是否能被目标领域读者检索到？
```

## 5.2 摘要 Abstract

### 写作目标

摘要应在有限篇幅中压缩完整研究链条：

```text
背景 / 问题
→ 缺口
→ 方法 / 实验 / 观测
→ 关键结果
→ 机制解释 / 科学贡献
→ 边界或意义
```

### 推荐结构

```text
1. 背景句：指出自然现象、材料体系、环境过程或生物机制的重要性。
2. 缺口句：指出已有研究未解决的机制、数据、尺度或条件问题。
3. 方法句：说明实验、观测、测量或表征方案。
4. 结果句：报告关键定量或定性结果。
5. 机制句：说明结果如何支持机制解释。
6. 贡献句：说明本文如何推进对该现象或体系的理解。
```

### 摘要写作要求

- 关键结果应尽量具体；
- 如果有定量结果，应给出关键数值、范围或方向；
- 不能只说“结果显著”；
- 方法不能只写“采用多种表征手段”；
- 贡献不能只写“为相关研究提供参考”；
- 机制判断必须与证据强度匹配。

### 示例骨架

```text
X 过程在 Y 系统中具有重要作用，但其在 Z 条件下的控制机制仍不清楚。
已有研究主要关注 A，却较少同时考察 B 与 C 的作用。
本文通过控制实验 / 现场观测 / 多尺度表征，测量了 D、E 和 F。
结果显示，在 G 条件下，X 增加 / 降低 / 转变了 H，并伴随 I 的变化。
结合对照实验和 J 表征结果，本文认为 K 机制可能解释上述现象。
这些结果表明，L 是调控 Y 系统行为的关键因素，并限定于本文测试的 M 条件范围。
```

### 常见错误

- 摘要写成背景介绍；
- 不出现具体结果；
- 只罗列方法，不提发现；
- 用“首次”“显著提高”“优异性能”但无数据支撑；
- 机制结论比正文证据更强；
- 摘要结论没有条件限定；
- 摘要中出现正文未报告的结果。

### Agent 质量判断

```text
1. 摘要是否明确科学问题？
2. 是否说明实验 / 观测 / 测量路径？
3. 是否报告关键结果，而非只说“研究发现”？
4. 是否说明这些结果意味着什么？
5. 是否避免过度声称？
6. 是否与正文图表和结论一致？
```

## 5.3 Introduction 引言

### 写作目标

自然科学论文的 Introduction 应从自然现象、机制问题或应用挑战出发，逐步引出一个具体、可用实验或观测回答的科学问题。

它的功能不是全面介绍领域，而是建立：

```text
为什么这个现象重要？
已有研究知道什么？
还不知道什么？
这个未知为什么值得研究？
本文如何通过实验 / 观测回答？
```

### 推荐组织方式

```text
1. 研究背景：说明现象、体系或过程的重要性。
2. 关键科学问题：指出当前理解中的机制、过程、尺度或条件问题。
3. 既有研究进展：概括已有发现，而不是逐篇罗列。
4. 研究缺口：说明仍缺少什么证据、机制解释或实验条件。
5. 本文研究问题 / 假设：明确本文要回答什么。
6. 研究策略：简要说明实验、观测、测量或表征路径。
7. 预期贡献：说明本文将如何推进理解。
```

### 引言中的常见问题类型

| 问题类型 | 示例 |
|---|---|
| 机制问题 | X 如何通过 Y 影响 Z？ |
| 控制因素问题 | 哪些因素限制 / 驱动某过程？ |
| 条件依赖问题 | 某机制在什么条件下成立？ |
| 尺度问题 | 微观结构如何影响宏观性能？ |
| 时间变化问题 | 某过程如何随季节、阶段或演化时间变化？ |
| 空间异质性问题 | 不同区域 / 深度 / 样点为何表现不同？ |
| 方法缺口问题 | 现有测量手段为何无法解析某过程？ |
| 反常现象问题 | 为什么观察结果与已有预期不一致？ |

### 好的缺口写法

弱缺口：

```text
目前关于 X 的研究还比较少。
```

强缺口：

```text
已有研究已证明 X 与 Y 存在相关关系，但尚未区分 A 机制和 B 机制的相对贡献，因此难以解释为什么在 Z 条件下会出现相反结果。
```

弱缺口：

```text
许多研究报道了该材料性能，但机制尚不清楚。
```

强缺口：

```text
虽然已有研究报道该材料在高温条件下具有较高反应活性，但多数研究仅比较终点性能，缺少对反应过程中中间相演化和表面价态变化的原位证据。
```

### 研究问题句式

```text
因此，本文关注的问题是：在 X 条件下，Y 如何影响 Z？
```

```text
本文旨在检验以下假设：A 通过改变 B 过程调控 C 响应。
```

```text
为区分 X 与 Y 两种可能机制，本文设计了对照实验并结合 Z 表征进行分析。
```

```text
本文通过长期现场观测，考察 A 沿 B 梯度的变化规律及其潜在驱动因素。
```

### 常见错误

- 从宏大背景写起，但迟迟不出现科学问题；
- 只说应用重要性，不说明科学未知；
- 文献综述按作者罗列；
- 缺口只是“研究较少”，没有机制意义；
- 引言末尾只说“本文进行了实验”，没有提出问题或假设；
- 引言承诺机制解释，但 Methods 和 Results 没有对应实验；
- 把技术路线写得过细，挤占问题提出；
- 引言与结果不闭合。

### Agent 质量判断

```text
1. 是否从现象或机制问题出发？
2. 是否指出已有研究已经知道什么？
3. 是否明确仍然不知道什么？
4. 缺口是否能用实验 / 观测回答？
5. 是否提出研究问题或假设？
6. 是否简要说明研究策略？
7. 是否预告贡献但不过度承诺？
```

## 5.4 Methods / Materials and Methods 方法

### 写作目标

Methods 的核心目标是支持：

```text
可重复性、可评价性、可解释性。
```

读者应能通过 Methods 判断：

- 实验或观测是如何设计的；
- 样品、材料、地点或对象从哪里来；
- 变量如何控制；
- 对照如何设置；
- 数据如何采集；
- 仪器如何校准；
- 重复和误差如何处理；
- 数据如何分析；
- 结论是否可信。

### Methods 应包含内容

#### 1. 研究对象 / 样品 / 材料

应说明：

- 样品来源；
- 样品制备过程；
- 样品编号或分组；
- 关键纯度、规格、批次或来源；
- 保存、处理和前处理方式；
- 纳入和排除标准。

#### 2. 实验设计 / 观测设计

应说明：

- 自变量是什么；
- 因变量是什么；
- 控制变量是什么；
- 对照组是什么；
- 处理条件是什么；
- 实验顺序是否随机；
- 是否有批次控制；
- 样本量和重复数；
- 时间、空间、温度、浓度、压力、光照、水分、pH 等关键条件。

#### 3. 实验步骤 / 采样步骤

应说明到足以让同行理解并复现：

- 操作顺序；
- 反应或处理时间；
- 温度、浓度、流速、转速、剂量等参数；
- 采样位置和频率；
- 样品保存条件；
- 数据采集时间点；
- 关键试剂、设备或传感器信息。

#### 4. 测量与表征方法

应说明：

- 使用何种仪器；
- 仪器型号和关键设置；
- 校准方式；
- 测量范围；
- 检测限或分辨率；
- 数据预处理方式；
- 峰识别、图像分析或信号处理标准；
- 是否使用标准样品或参考物质。

#### 5. 统计分析与不确定性处理

应说明：

- 重复类型：技术重复、生物重复、独立重复、批次重复；
- 样本量 n；
- 误差表示方式：标准差、标准误、置信区间、四分位范围等；
- 统计检验方法；
- 检验前提；
- 显著性阈值；
- 多重比较处理；
- 异常值处理规则；
- 软件和版本，如有必要；
- 数据排除标准。

#### 6. 伦理、许可与安全

根据领域需要，应说明：

- 动物、植物、微生物、野外采样、人源样本等是否涉及审批或许可；
- 采样是否获得当地许可；
- 生物安全、化学安全或辐射安全是否按要求处理；
- 具体要求应根据机构、期刊和研究对象执行，不能虚构规范条款。

### Methods 的写作原则

| 原则 | 含义 |
|---|---|
| 充分但不冗余 | 写出影响结果解释的关键信息 |
| 可重复 | 同行可据此理解如何再做一次 |
| 可评价 | 审稿人能判断设计是否合理 |
| 与结果对应 | Results 中每个关键数据都能在 Methods 找到来源 |
| 不隐藏选择 | 样品筛选、数据排除和异常值处理要透明 |
| 不提前讨论 | Methods 不应解释结果意义 |

### 变量、对照、重复、误差在写作中的体现

#### 变量

写法：

```text
The concentration of X was varied from A to B while temperature, pH, and reaction time were held constant.
```

中文表达：

```text
本文将 X 浓度设为 A—B 范围，并保持温度、pH 和反应时间不变，以考察 X 对 Y 的影响。
```

Agent 检查：

```text
1. 自变量是否清楚？
2. 因变量是否清楚？
3. 控制变量是否清楚？
4. 是否存在未说明的干扰因素？
```

#### 对照

常见对照类型：

| 对照类型 | 功能 |
|---|---|
| 空白对照 | 判断背景信号或自然变化 |
| 阴性对照 | 判断没有关键因素时是否无效应 |
| 阳性对照 | 判断系统是否能检测到预期效应 |
| 基准样品 | 与已有材料、标准材料或商业样品比较 |
| 未处理组 | 判断处理是否产生变化 |
| 时间零点 | 判断过程随时间变化 |
| 同位素 / 标记对照 | 判断来源或路径 |
| 环境背景样点 | 判断自然背景水平 |

Agent 检查：

```text
没有对照时，很多“效果”只能是观察，不能作为因果解释。
```

#### 重复

应区分：

| 类型 | 含义 | 写作作用 |
|---|---|---|
| 技术重复 | 同一样品重复测量 | 估计测量误差 |
| 生物重复 | 不同生物个体或独立生物样本 | 估计生物变异 |
| 独立实验重复 | 独立制备 / 独立实验批次 | 支持可再现性 |
| 空间重复 | 不同样点或区域 | 支持空间代表性 |
| 时间重复 | 不同时间点或季节 | 支持时间稳定性 |

常见错误：

- 把技术重复当作生物重复；
- 只报告平均值，不报告 n；
- 只做一次实验却写成普遍规律；
- 不说明误差条代表什么。

#### 误差与不确定性

写作中应说明：

- 误差条代表 SD、SEM、CI 还是其他；
- 测量不确定性来源；
- 仪器检测限；
- 标准曲线或校准结果；
- 数据处理造成的不确定性；
- 观察型研究中的空间或时间不确定性。

#### 统计检验

写作中应避免：

- 只写 “significant” 而不说明检验；
- p 值替代效应大小；
- 对非独立数据使用独立样本检验；
- 忽略多重比较；
- 不说明 n；
- 将统计显著性等同于科学重要性。

### Methods 常见错误

- 只写“采用常规方法”；
- 关键参数缺失；
- 没有说明对照；
- 没有说明重复；
- 没有说明样品筛选；
- 仪器参数不足；
- 数据处理流程不透明；
- 统计方法与数据类型不匹配；
- 把 Results 写进 Methods；
- 无法追溯每张图的数据来源。

### Agent 质量判断

```text
1. 读者是否知道样品 / 数据从哪里来？
2. 读者是否知道实验如何被执行？
3. 是否说明了自变量、因变量和控制变量？
4. 是否有足够对照？
5. 是否报告重复数和误差处理方式？
6. 仪器和测量参数是否足够？
7. 统计方法是否与数据结构匹配？
8. Methods 是否支撑 Results 中所有关键图表？
```

## 5.5 Results 结果

### 写作目标

Results 的任务是呈现实验、观测和测量所产生的核心证据。  
它应回答：

```text
数据展示了什么？
图表中的主要模式是什么？
这些结果支持哪个中间发现？
下一步为什么需要另一个实验或分析？
```

Results 不是实验日志，也不是把所有数据按实验时间顺序贴出来。

### Results 的组织原则

#### 1. 按科学问题组织

适合研究问题明确的论文。

```text
3.1 X 处理改变了 Y 的基础特征
3.2 Y 的变化与 Z 性能提升相关
3.3 对照实验排除了 A 解释
3.4 原位表征支持 B 机制
```

#### 2. 按证据链组织

适合机制解释或多表征论文。

```text
现象确认
→ 结构 / 成分证据
→ 性能 / 响应证据
→ 动态过程证据
→ 机制验证证据
```

#### 3. 按尺度组织

适合材料、地球、生态或生物系统。

```text
宏观响应
→ 微观结构
→ 分子 / 化学机制
→ 系统水平影响
```

#### 4. 按时间 / 空间梯度组织

适合观察型论文。

```text
时间变化
→ 空间分布
→ 驱动因子
→ 异常区域或阶段
```

#### 5. 按假设组织

适合假设驱动实验。

```text
H1：X 影响 Y
H2：该影响依赖 Z
H3：该影响通过 M 机制发生
```

### Results 段落结构

推荐结构：

```text
1. 小主张：本节要显示什么发现。
2. 数据呈现：描述图表、数值、趋势、差异或表征结果。
3. 关键比较：与对照、基准、时间点或梯度比较。
4. 结果解释限度：只说明直接结果，不展开过度机制讨论。
5. 过渡句：说明下一步分析为什么必要。
```

示例骨架：

```text
如图 2a 所示，X 处理后样品的 Y 信号明显增强，而对照组未观察到类似变化。定量分析显示，Y 的平均值由 A 增至 B，且在三次独立重复中保持一致。该结果表明，X 处理能够稳定改变 Y 特征。为进一步判断这种变化是否与 Z 性能相关，本文随后进行了……
```

### 图表叙事

自然科学论文常由图表构成“发现链条”。  
每张主图都应承担明确功能，而不是装饰或堆数据。

#### 图表功能链示例

| 图号 | 功能 | 应支持的结论 |
|---|---|---|
| Figure 1 | 研究设计 / 样品构建 / 观测区域 | 说明研究对象和实验逻辑 |
| Figure 2 | 基础表征 / 初始观测 | 确认系统差异或现象存在 |
| Figure 3 | 性能 / 响应 / 变量关系 | 证明关键效应 |
| Figure 4 | 对照 / 稳健性 / 参数分析 | 排除替代解释 |
| Figure 5 | 机制证据 | 支持机制解释 |
| Figure 6 | 模型 / 综合示意 | 总结发现链条 |

#### 每张图应回答四个问题

```text
1. 这张图测量了什么？
2. 最重要的趋势或差异是什么？
3. 它支持哪个中间结论？
4. 它如何连接下一张图或最终结论？
```

### 图表文字写法

弱写法：

```text
Figure 2 shows the SEM images of the samples.
```

较好写法：

```text
Figure 2 compares the surface morphology of untreated and X-treated samples, showing that X treatment produces a more porous structure with increased surface roughness.
```

弱写法：

```text
The result is shown in Fig. 3.
```

较好写法：

```text
As shown in Fig. 3, the reaction rate increased with temperature up to 80 °C and then declined, indicating that the process is limited by thermal deactivation at higher temperatures.
```

### Results 中应如何处理表征结果

表征结果不能只写“进行了 SEM、XRD、FTIR、XPS、TEM、Raman、GC-MS、qPCR、Western blot 等分析”。  
应说明每种表征在证据链中的功能。

示例：

| 表征 / 测量 | 不好的写法 | 好的写法 |
|---|---|---|
| SEM / TEM | 观察形貌 | 证明处理改变了微观结构 |
| XRD | 分析晶相 | 确认新相形成或晶体结构保持 |
| XPS | 分析价态 | 支持表面电子结构变化 |
| FTIR / Raman | 分析官能团 | 证明化学键或分子环境变化 |
| UV-vis / PL | 光谱分析 | 判断吸收、跃迁或复合过程 |
| qPCR / Western blot | 检测表达 | 证明分子通路被激活或抑制 |
| 同位素示踪 | 追踪来源 | 支持物质转化路径 |
| 遥感 / 现场监测 | 观察时空变化 | 识别自然系统模式 |

### Results 与 Discussion 的边界

Results 应主要回答：

```text
观察到什么？
测量到什么？
差异是什么？
趋势是什么？
统计结果是什么？
```

Discussion 应主要回答：

```text
为什么会这样？
这与已有研究有什么关系？
这说明什么机制？
有哪些替代解释？
结论边界是什么？
```

边界示例：

Results 中可以写：

```text
X 处理组的 Y 值显著高于对照组。
```

Discussion 中再写：

```text
这种差异可能源于 X 处理诱导的表面缺陷增加，因为缺陷位点可促进……
```

Results 中可以写：

```text
在高海拔样点，土壤有机碳含量随深度下降更快。
```

Discussion 中再写：

```text
这一垂直梯度可能反映了低温条件下表层植物残体输入与深层分解受限之间的共同作用。
```

### 常见错误

- 按实验时间顺序写成流水账；
- 每张图只描述“如图所示”，没有提出发现；
- 图表过多但缺少主线；
- 只呈现平均值，不报告误差和 n；
- 不说明统计检验；
- 用 Results 做过度机制推断；
- 不处理异常点或反常结果；
- 结果和方法重复；
- 结果小节标题过于笼统，如“表征分析”“性能测试”“数据分析”；
- 所有结果等量呈现，没有突出关键证据。

### Agent 质量判断

```text
1. Results 是否按证据链组织？
2. 每个小节是否有明确中间发现？
3. 每张图是否承担具体论证功能？
4. 是否报告对照、重复、误差和统计信息？
5. 数据描述是否足够具体？
6. 是否避免把结果写成实验记录？
7. 是否为 Discussion 的机制解释提供证据基础？
```

## 5.6 Discussion 讨论

### 写作目标

Discussion 的任务是把 Results 中的事实提升为科学解释。  
它应回答：

```text
这些结果为什么出现？
它们支持什么机制？
它们与已有研究一致还是冲突？
异常结果如何解释？
还有哪些替代解释不能排除？
结论适用于哪些条件？
```

### 推荐组织方式

```text
1. 回扣核心发现
2. 解释可能机制
3. 与已有研究比较
4. 处理异常结果或不一致证据
5. 讨论不确定性和局限性
6. 说明科学贡献和适用边界
```

### Discussion 的主要功能

| 功能 | 写作重点 |
|---|---|
| 机制解释 | 说明多个结果如何共同支持某机制 |
| 文献比较 | 解释为何与已有研究一致或不同 |
| 异常处理 | 不回避反常数据，提出合理解释 |
| 替代解释 | 说明哪些解释已排除，哪些仍可能存在 |
| 边界限定 | 明确结论适用条件 |
| 意义提升 | 说明对理论、机制、方法或应用的贡献 |

### 机制解释写法

弱写法：

```text
这些结果说明 X 具有良好性能。
```

强写法：

```text
这些结果表明，X 的性能提升并非仅由比表面积增加导致。首先，A 对照样品具有相似比表面积但性能较低；其次，原位光谱显示反应过程中 B 中间体更稳定；最后，C 处理削弱了该中间体信号并降低了反应速率。因此，本文认为 B 中间体稳定化是 X 提高性能的重要机制。
```

### 文献比较写法

弱写法：

```text
本研究结果与前人研究一致。
```

强写法：

```text
本研究观察到 X 随温度升高而增加，这与已有关于低温限制 Y 过程的研究一致。但与仅强调温度作用的解释不同，本文进一步显示，在高湿度条件下该关系明显减弱，说明水分状态可能调节温度效应。
```

### 异常结果处理

异常结果不应被忽略。可按以下逻辑处理：

```text
1. 明确指出异常或不一致结果；
2. 检查是否可能来自测量误差或样本问题；
3. 与对照、重复或补充实验比较；
4. 提出可能机制解释；
5. 说明当前数据能支持到什么程度；
6. 将未解决部分作为限制或未来研究。
```

示例：

```text
值得注意的是，最高处理浓度下的响应并未继续增强，而是出现下降。这一结果可能与高浓度导致的结构塌陷或活性位点阻塞有关。虽然本文的形貌表征支持结构变化这一解释，但尚不能完全排除溶液传质受限的影响。因此，未来需要结合原位传质测量进一步验证。
```

### 结论强度分级

Discussion 中应根据证据强度选择表达。

| 证据情况 | 合适表达 | 不合适表达 |
|---|---|---|
| 有直接实验、对照和重复 | 结果表明 / 支持 | 彻底证明 |
| 有相关观测但无操控实验 | 结果提示 / 可能相关 | 导致 / 决定 |
| 有间接表征 | 可能说明 / 与……一致 | 证明机制是 |
| 只有单一实验 | 初步表明 | 普遍适用 |
| 有模型或推测 | 推测 / 假设 | 已证实 |

### Discussion 常见错误

- 只是重复 Results；
- 每段都以“这说明……”开头，但证据不足；
- 不处理与已有研究相反的结果；
- 不讨论方法限制；
- 将相关关系写成因果关系；
- 机制解释只靠一张图；
- 用“可能是由于”提出很多推测，但没有证据分级；
- 结论推广到未测试条件；
- 不说明实验条件边界。

### Agent 质量判断

```text
1. Discussion 是否解释了核心发现？
2. 是否把多个结果连接为机制链条？
3. 是否与关键文献对话？
4. 是否处理异常结果？
5. 是否区分证据支持、合理推测和未来假设？
6. 是否限定适用条件？
7. 是否避免把 Discussion 写成 Results 的重复版本？
```

## 5.7 Conclusion 结论

### 写作目标

Conclusion 应简洁回答科学问题，压缩核心证据链和贡献，并明确结论边界。

### 推荐结构

```text
1. 回到研究问题。
2. 概括实验 / 观测路径。
3. 总结核心发现。
4. 提炼机制或科学解释。
5. 说明贡献。
6. 限定适用条件。
7. 指出未来工作。
```

### 示例骨架

```text
本文围绕 X 在 Y 条件下如何影响 Z 的问题，结合控制实验、表征分析和对照验证进行了研究。
结果显示，X 处理显著改变了 A，并进一步影响 B 响应。
多组证据表明，该过程可能通过 C 机制发生，而不是单纯由 D 因素解释。
这些发现为理解 Y 系统中的 Z 变化提供了新的实验证据。
需要指出的是，本文结论主要适用于 E 条件，未来仍需在 F 场景中进一步验证。
```

### 常见错误

- 只重复摘要；
- 写成“本文完成了若干实验”；
- 只强调应用价值，不回答科学问题；
- 引入新数据；
- 结论比 Discussion 更夸张；
- 不说明适用边界；
- 未来研究与本文限制无关。

### Agent 质量判断

```text
1. 是否回答 Introduction 提出的问题？
2. 是否总结最关键发现，而不是罗列所有结果？
3. 是否说明机制解释的证据基础？
4. 是否明确贡献？
5. 是否有边界限定？
6. 是否避免新证据和过度声称？
```

## 6. 常见贡献类型

实验 / 观察型自然科学论文的贡献通常不是单一的“性能提升”或“发现现象”，而是某种可被学术共同体吸收的新知识。

### 6.1 现象发现贡献

发现此前未被充分观察、测量或描述的自然现象。

示例：

```text
本文在长期监测数据中发现 X 过程存在明显季节性反转，提示该系统对 Y 条件变化具有非线性响应。
```

质量要求：

- 现象必须由可靠观测支持；
- 需要说明为什么不是测量误差或偶然事件；
- 需要说明该现象为何科学上重要。

### 6.2 机制解释贡献

揭示某现象、反应、过程、性能或生态响应背后的机制。

示例：

```text
本文通过原位表征和对照实验表明，X 的增强主要源于 Y 中间体稳定化，而不是总活性位点数量增加。
```

质量要求：

- 需要多组证据；
- 需要对照或替代解释排除；
- 不能只依赖单一相关结果。

### 6.3 方法 / 测量贡献

提出或改进实验、观测、表征或测量方法。

示例：

```text
本文建立了一种低扰动原位测量方法，使反应过程中 X 的瞬态变化能够被连续追踪。
```

质量要求：

- 说明方法解决什么测量难题；
- 与已有方法比较；
- 报告精度、检测限、重复性或适用范围；
- 展示方法如何带来新知识。

### 6.4 数据贡献

提供新的数据集、长期观测记录、区域样点数据或系统测量结果。

示例：

```text
本文提供了某流域连续五年的高频营养盐通量数据，揭示了极端降雨事件对年际输出的非线性影响。
```

质量要求：

- 数据来源透明；
- 采样设计合理；
- 质量控制清楚；
- 数据用于回答科学问题，而不是单纯发布数据。

### 6.5 材料 / 样品贡献

合成、制备或发现具有特定结构和功能的新材料、样品或体系。

示例：

```text
本文制备的 X 材料在保持结构稳定性的同时提高了 Y 性能，并通过 Z 表征解释了性能提升来源。
```

质量要求：

- 不能只说“新材料”；
- 要说明结构、性能和机制之间的关系；
- 需要与基准或已有材料比较；
- 稳定性和重复性要充分报告。

### 6.6 过程规律贡献

揭示变量之间的规律、阈值、梯度或阶段性变化。

示例：

```text
本文发现 X 过程对温度的响应存在阈值效应，在 A 范围内增强，而超过 B 后转为抑制。
```

质量要求：

- 变量范围和条件要清楚；
- 需要足够数据点；
- 需要讨论阈值是否具有机制意义；
- 不能把有限范围外推到所有条件。

### 6.7 反常现象解释贡献

解释与已有理论或经验预期不一致的结果。

示例：

```text
本文解释了为什么在高湿度条件下，通常促进 X 的温度升高反而削弱 Y 过程。
```

质量要求：

- 明确反常在哪里；
- 证明不是实验误差；
- 提供可检验解释；
- 与已有研究形成对话。

### 6.8 边界条件贡献

说明某已知机制、材料性能或自然规律在什么条件下成立或失效。

示例：

```text
本文表明，X 机制仅在低离子强度条件下主导反应过程，而在高离子强度条件下被 Y 过程取代。
```

质量要求：

- 条件范围明确；
- 对比充分；
- 讨论理论或实践意义。

## 7. 证据与论证标准

## 7.1 强证据的特征

自然科学论文中的强证据通常具有以下特征：

```text
1. 直接对应研究问题；
2. 有合理对照；
3. 有重复和误差估计；
4. 测量方法可靠；
5. 数据处理透明；
6. 统计分析合适；
7. 能排除关键替代解释；
8. 多组证据收敛；
9. 结论被限定在数据支持范围内。
```

### 强证据示例

| 主张 | 强证据组合 |
|---|---|
| X 提高材料性能 | 性能测试 + 对照样品 + 重复 + 误差条 + 稳定性 + 结构表征 |
| X 通过 Y 机制发挥作用 | 对照实验 + 原位表征 + 中间体检测 + 抑制 / 增强实验 |
| X 与环境梯度相关 | 多样点采样 + 时间重复 + 混杂因素分析 + 统计模型 |
| 某观测现象真实存在 | 独立仪器验证 + 校准 + 重复观测 + 不确定性估计 |
| 某处理影响生物响应 | 合理对照 + 生物重复 + 统计检验 + 分子或生理指标支持 |

## 7.2 弱证据的特征

弱证据常表现为：

- 没有对照；
- 没有重复；
- 没有误差条；
- 未说明 n；
- 只展示代表性图片；
- 只报告最优结果；
- 只用单一表征支撑机制；
- 相关关系写成因果；
- 对异常值不解释；
- 方法参数不足；
- 选择性报告数据；
- 结论推广到未测试体系；
- 统计方法不适合数据结构。

## 7.3 变量、对照、重复、误差和统计的写作标准

### 变量

论文中应明确：

```text
自变量：作者操控或比较的因素。
因变量：作者测量的响应。
控制变量：需要保持一致的条件。
混杂变量：可能影响解释但未完全控制的因素。
```

Agent 判断：

```text
如果读者无法说出“改变了什么、测量了什么、控制了什么”，实验设计就不清楚。
```

### 对照

对照的功能是支撑解释，而不只是形式要求。

Agent 应检查：

```text
1. 对照是否与研究问题对应？
2. 对照能否排除背景效应？
3. 对照能否区分两个竞争机制？
4. 是否需要阳性或阴性对照？
5. 是否需要基准材料或标准样品？
```

### 重复

写作中必须避免模糊表述：

弱写法：

```text
All experiments were repeated.
```

较好写法：

```text
Each experiment was performed using three independently prepared samples, and each sample was measured in triplicate.
```

中文表达：

```text
每组实验包括三个独立制备样品，每个样品进行三次技术重复测量。
```

### 误差

Agent 应检查：

```text
1. 图中误差条代表什么？
2. 是否报告 n？
3. 是否报告仪器检测限或测量不确定性？
4. 是否说明数据排除标准？
5. 是否区分自然变异和测量误差？
```

### 统计检验

自然科学写作中，统计检验应服务于科学问题，而不是装饰结果。

应说明：

- 数据是否独立；
- 是否符合检验假设；
- 检验方法为何合适；
- 显著性阈值；
- 效应大小；
- 多重比较；
- 对小样本的谨慎解释。

## 7.4 结果强度与表达强度匹配

| 数据支持程度 | 建议表达 | 避免表达 |
|---|---|---|
| 多实验、多对照、重复充分 | 结果表明 / 支持 | 彻底证明 |
| 有明确差异但机制未验证 | 可能与……有关 | 机制是…… |
| 有相关性但无操控 | 与……相关 / 提示 | 导致 |
| 单一实验或初步数据 | 初步结果显示 | 普遍规律 |
| 观察型数据 | 可能受……影响 | 由……决定 |
| 模型辅助解释 | 与……机制一致 | 已证实机制 |

## 7.5 论证链质量判断

Agent 可使用以下判断表：

| 问题 | 诊断 | 修正 |
|---|---|---|
| 结论超出数据 | 是否把相关写成因果？ | 改为限定表达 |
| 机制证据不足 | 是否只有表征结果？ | 增加对照或改为“可能机制” |
| 对照缺失 | 是否无法排除背景效应？ | 补写对照逻辑或降低结论强度 |
| 重复不足 | 是否没有 n 和误差？ | 补充重复信息或限定结果 |
| 图表孤立 | 每张图是否连接主线？ | 重排图表叙事 |
| 异常未解释 | 是否忽略反常数据？ | 增加异常分析 |
| 方法不透明 | 是否无法复现？ | 补充关键参数 |

## 8. 文献综述写法

### 8.1 自然科学文献综述的功能

在实验 / 观察型自然科学论文中，文献综述通常嵌入 Introduction 和 Discussion。其功能不是罗列前人做了什么，而是建立：

```text
现象背景
→ 已有机制解释
→ 证据不足
→ 方法或尺度限制
→ 本文实验 / 观测的必要性
```

### 8.2 文献应如何组织

#### 方式一：机制路径组织

适合机制解释论文。

```text
已有研究提出机制 A。
另一些研究支持机制 B。
但在 X 条件下，两种机制的相对作用尚未区分。
本文通过对照实验和原位表征检验……
```

#### 方式二：尺度组织

适合跨尺度研究。

```text
宏观研究已发现 X 现象。
微观研究揭示了 Y 过程。
但二者之间的连接机制仍不清楚。
本文结合宏观测量与微观表征……
```

#### 方式三：方法限制组织

适合测量、表征或观测方法论文。

```text
传统方法能够测量终点状态。
近期方法可捕捉部分动态过程。
但现有方法仍难以在原位条件下解析 X。
本文采用……
```

#### 方式四：条件差异组织

适合边界条件研究。

```text
已有研究多在 A 条件下进行。
但在 B 条件下，系统可能表现出不同响应。
本文考察……
```

#### 方式五：观察矛盾组织

适合解释反常现象。

```text
部分研究认为 X 促进 Y。
另一些研究观察到 X 抑制 Y。
这种分歧可能源于 Z 条件差异。
本文通过……
```

### 8.3 文献综述段落结构

推荐结构：

```text
主题句：说明这一组研究关注什么。
综合句：概括已有研究共同发现。
不足句：指出未解决的机制、证据、尺度或条件问题。
转入句：说明本文如何回应这个不足。
```

示例：

```text
已有研究普遍认为表面缺陷是影响该类催化材料活性的关键因素。相关实验显示，缺陷浓度增加通常伴随反应速率提高。然而，多数研究主要基于反应前后的静态表征，难以判断缺陷在反应过程中的真实作用。为弥补这一不足，本文结合原位光谱和缺陷钝化对照实验，考察缺陷位点对中间体稳定性的影响。
```

### 8.4 常见文献缺口类型

| 缺口类型 | 含义 | 好写法 |
|---|---|---|
| 机制缺口 | 知道现象，不知道原因 | 尚不清楚 X 通过何种路径影响 Y |
| 过程缺口 | 只知道终点，不知道过程 | 缺少对反应 / 演化过程的动态证据 |
| 尺度缺口 | 不同尺度未连接 | 宏观性能与微观结构之间关系未明 |
| 条件缺口 | 某条件下未验证 | 既有结论是否适用于 Z 条件仍未知 |
| 方法缺口 | 现有技术不足 | 难以原位捕捉 X 的瞬态变化 |
| 数据缺口 | 缺少长期 / 区域 / 高分辨率数据 | 缺少跨季节连续观测 |
| 对照缺口 | 竞争解释未区分 | 尚未排除 A 与 B 两种机制 |
| 不确定性缺口 | 误差和变异未处理 | 现有研究较少量化测量不确定性 |

### 8.5 常见错误

- 文献综述写成“某某做了什么”；
- 只引用支持自己结论的文献；
- 忽略与结果相反的研究；
- 不说明已有研究的实验条件；
- 不区分直接证据和推测；
- 用“尚未有人研究”替代科学缺口；
- 引言中的文献缺口与 Discussion 中的贡献不对应；
- 综述不导向本文实验设计。

### Agent 质量判断

```text
1. 文献是否围绕科学问题组织？
2. 是否说明已有机制解释？
3. 是否指出具体证据不足？
4. 是否说明本文实验 / 观测为何必要？
5. 是否处理相反结果或竞争机制？
6. 是否自然导出研究假设或实验设计？
```

## 9. 结果、分析或论证主体的组织方式

## 9.1 主体不是实验流水账

弱结构：

```text
3.1 SEM 结果
3.2 XRD 结果
3.3 FTIR 结果
3.4 性能测试
3.5 机理分析
```

这种结构的问题是：

- 按技术手段堆叠；
- 读者难以看出科学问题；
- 每种结果孤立；
- 图表之间没有发现链条。

强结构：

```text
3.1 X 处理诱导了稳定的结构重构
3.2 结构重构显著提升了 Y 响应
3.3 对照实验排除了表面积效应的主导作用
3.4 原位表征揭示了 Z 中间体稳定化过程
3.5 多组证据支持“结构—中间体—性能”机制链条
```

### 9.2 图表叙事的核心原则

图表顺序应服务于发现链条，而不是服务于实验完成顺序。

推荐图表逻辑：

```text
Figure 1：问题与研究设计
Figure 2：样品 / 现象基础确认
Figure 3：关键效应或主要结果
Figure 4：变量关系和对照验证
Figure 5：机制证据
Figure 6：综合模型或边界验证
```

### 9.3 每个结果小节的功能

每个结果小节都应有一个明确小结论。

弱标题：

```text
3.2 表征结果
```

强标题：

```text
3.2 X 处理改变了表面氧化态而未破坏主体晶相
```

弱标题：

```text
3.3 性能测试
```

强标题：

```text
3.3 表面氧化态变化与反应速率提高相对应
```

弱标题：

```text
3.4 影响因素分析
```

强标题：

```text
3.4 pH 升高削弱了 X 对 Y 的促进作用
```

### 9.4 从数据中提炼故事线

Agent 可按以下步骤帮助用户提炼结果故事线：

```text
1. 列出所有实验 / 图表。
2. 给每个实验标注功能：确认、比较、排除、解释、验证。
3. 找出最核心科学问题。
4. 删除或降级与问题无关的图表。
5. 将图表按逻辑递进重排。
6. 为每个图表生成一句中间发现。
7. 将中间发现连接成完整证据链。
```

示例：

```text
原始实验列表：
- SEM
- XRD
- BET
- XPS
- 性能测试
- 稳定性测试
- 抑制剂实验
- 原位光谱

重构后的故事线：
1. X 处理没有改变主体晶相，但改变了表面结构。
2. 表面结构变化提高了目标反应性能。
3. 性能提升不能仅由比表面积解释。
4. 原位光谱显示关键中间体更稳定。
5. 抑制剂实验进一步支持该中间体参与机制。
6. 因此，X 处理通过调节表面反应路径提高性能。
```

### 9.5 处理补充数据

不是所有数据都应进入主文。

主文图表应满足：

```text
1. 直接支持核心结论；
2. 构成发现链条中的关键节点；
3. 对理解机制或设计必要；
4. 审稿人会要求看到。
```

补充材料适合放置：

- 方法验证；
- 额外重复；
- 额外参数；
- 非核心表征；
- 原始谱图；
- 稳健性检查；
- 详细采样点信息；
- 额外模型拟合。

### 9.6 观察型论文主体组织

观察型自然科学论文应特别注意空间、时间和尺度逻辑。

常见组织方式：

#### 按时间

```text
季节变化
→ 年际变化
→ 极端事件响应
→ 长期趋势
```

#### 按空间

```text
区域差异
→ 梯度变化
→ 热点区域
→ 空间驱动因素
```

#### 按过程

```text
输入变化
→ 转化过程
→ 输出响应
→ 系统反馈
```

#### 按变量关系

```text
X 与 Y 的相关模式
→ 控制变量后的关系
→ 不同情境下的异质性
→ 可能机制解释
```

### 9.7 实验型论文主体组织

实验型论文应体现“从效应到机制”的递进。

常见组织方式：

```text
1. 样品 / 体系确认
2. 处理或变量产生主要效应
3. 效应随条件变化
4. 对照排除替代解释
5. 表征或检测揭示机制
6. 稳定性 / 泛化性 / 边界检验
```

## 10. 讨论与结论写法

## 10.1 Discussion 与 Results 的边界

| 内容 | 放在 Results | 放在 Discussion |
|---|---|---|
| 图中趋势 | 是 | 可简要引用 |
| 关键数值 | 是 | 只引用最关键数值 |
| 统计显著性 | 是 | 解释其科学意义 |
| 表征峰位置变化 | 是 | 解释其机制含义 |
| 与文献比较 | 少量必要 | 重点展开 |
| 机制模型 | 初步引出 | 系统论证 |
| 异常解释 | 简要指出 | 深入讨论 |
| 局限性 | 通常不展开 | 必须讨论 |
| 未来研究 | 不写 | 可写 |

### 10.2 Discussion 的推荐逻辑

```text
核心发现是什么？
→ 它如何回答研究问题？
→ 哪些证据共同支持该解释？
→ 与已有研究相比新在哪里？
→ 异常和不确定性如何理解？
→ 结论在哪些条件下成立？
```

### 10.3 从结果到机制的写法

结果层：

```text
X 处理组的 Y 指标高于对照组。
```

机制层：

```text
该差异与 Z 表征结果显示的表面缺陷增加一致。
```

论证层：

```text
由于缺陷钝化实验显著削弱了 Y 响应，本文认为表面缺陷在 X 诱导的性能提升中发挥关键作用。
```

边界层：

```text
不过，该机制目前仅在本文测试的温度和反应体系中得到验证。
```

### 10.4 如何比较文献

比较文献时不要只写“与前人一致”。应说明：

```text
1. 一致在哪里？
2. 不一致在哪里？
3. 为什么会不同？
4. 本文提供了什么新证据？
5. 本文是否改变已有解释的适用范围？
```

示例：

```text
与已有研究中 X 在低温下促进 Y 的结论一致，本文也观察到类似趋势。然而，本文进一步发现，当水分含量超过某一阈值后，该促进作用减弱。这表明温度效应可能受到水分条件调节，从而解释了不同区域研究结果之间的差异。
```

### 10.5 如何限定结论

自然科学论文的结论必须与实验 / 观测范围一致。

弱写法：

```text
X 是控制 Y 的关键因素。
```

较好写法：

```text
在本文测试的 A—B 条件范围内，X 是影响 Y 变化的主要因素之一。
```

弱写法：

```text
该材料可广泛应用于实际环境。
```

较好写法：

```text
该材料在实验室条件下表现出较高稳定性，但其在复杂实际环境中的长期表现仍需进一步验证。
```

弱写法：

```text
这些结果证明了机制 M。
```

较好写法：

```text
这些结果支持机制 M，但尚不能排除 N 过程在特定条件下的贡献。
```

### 10.6 Conclusion 写作

Conclusion 不应重复所有结果，而应用最少语言回答：

```text
本文解决了什么科学问题？
最关键证据是什么？
形成了什么机制解释或规律认识？
贡献是什么？
边界是什么？
```

推荐结论段落：

```text
本文通过……研究了……问题。结果表明……。结合……和……证据，本文认为……。这一发现说明……，为……提供了新的实验 / 观测依据。由于本文实验 / 观测主要限于……条件，未来仍需在……中进一步检验。
```

## 11. 审稿人关注点

实验 / 观察型自然科学论文的审稿人通常会从以下方面质疑论文。

### 11.1 科学问题是否清楚

审稿人会问：

```text
1. 本文到底要回答什么科学问题？
2. 这个问题是机制问题、现象问题、方法问题还是应用问题？
3. 引言是否说明了为什么这个问题值得研究？
4. 实验设计是否围绕该问题？
```

### 11.2 实验设计是否合理

审稿人会问：

```text
1. 自变量、因变量和控制变量是否明确？
2. 是否有必要对照？
3. 对照是否能排除替代解释？
4. 样本量和重复是否足够？
5. 实验条件是否合理？
6. 是否存在批次效应？
7. 是否需要随机化或盲法？
8. 是否存在未控制的混杂因素？
```

### 11.3 数据质量是否可信

审稿人会问：

```text
1. 数据来源是否清楚？
2. 仪器是否校准？
3. 测量误差是否报告？
4. 图表是否包含 n 和误差条？
5. 是否只展示代表性图片？
6. 是否有原始数据或补充数据？
7. 数据处理是否透明？
8. 是否存在选择性报告？
```

### 11.4 统计分析是否合适

审稿人会问：

```text
1. 统计检验是否适合数据类型？
2. 样本是否独立？
3. n 是否足够？
4. 是否说明误差条含义？
5. 是否处理多重比较？
6. 是否报告效应大小？
7. 显著性是否被过度解释？
```

### 11.5 机制解释是否充分

审稿人会问：

```text
1. 机制解释是否由直接证据支持？
2. 是否只是相关性推断？
3. 是否有对照实验区分竞争机制？
4. 是否有原位、动态或路径证据？
5. 是否排除了更简单解释？
6. 是否承认机制证据的间接性？
```

### 11.6 结论强度是否合适

审稿人会问：

```text
1. 结论是否超过数据范围？
2. 是否把实验室结果推广到实际环境？
3. 是否把短期结果推广到长期稳定性？
4. 是否把单一物种 / 样品 / 区域推广到普遍规律？
5. 是否把相关关系写成因果关系？
6. 是否使用过度宣传语言？
```

### 11.7 图表是否支撑故事线

审稿人会问：

```text
1. 主图是否都是必要的？
2. 图表顺序是否形成发现链条？
3. 图注是否清楚？
4. 图中单位、样本量、误差条是否明确？
5. 图表是否重复？
6. 是否缺少关键对照图？
```

### 11.8 方法是否可重复

审稿人会问：

```text
1. 关键实验参数是否完整？
2. 样品制备是否足够清楚？
3. 采样地点、时间和条件是否明确？
4. 仪器设置是否可复现？
5. 数据处理流程是否透明？
6. 是否需要在补充材料中提供更多细节？
```

## 12. 常见失败模式

## 12.1 把论文写成实验记录流水账

表现：

```text
先做了 A，然后做了 B，接着做了 C，最后做了 D。
```

问题：

- 读者不知道科学问题；
- 实验之间缺少逻辑；
- 结果不能自然导向结论。

修正：

```text
1. 明确核心科学问题。
2. 给每个实验分配论证功能。
3. 按证据链重排结果。
4. 用发现式小标题替代技术式小标题。
```

## 12.2 数据很多，但没有主线

表现：

- 图很多；
- 表征很多；
- 每张图都描述了，但没有中心结论。

修正：

```text
1. 确定主结论。
2. 标注每张图支持哪个中间结论。
3. 删除或移入补充材料的次要图。
4. 让图表顺序形成递进。
```

## 12.3 缺少对照

表现：

- 只有处理组；
- 没有空白或未处理组；
- 没有基准样品；
- 无法判断效应是否由目标因素引起。

修正：

```text
1. 增加必要对照。
2. 如果无法补实验，则降低结论强度。
3. 在 Discussion 中承认替代解释。
4. 避免因果表达。
```

## 12.4 重复和误差报告不足

表现：

- 不写 n；
- 不写误差条含义；
- 只展示代表性图像；
- 单次实验得出强结论。

修正：

```text
1. 补充重复数。
2. 说明误差条含义。
3. 区分技术重复和独立重复。
4. 增加统计说明。
5. 限定结论范围。
```

## 12.5 机制解释过度

表现：

- 一个光谱峰变化就声称证明机制；
- 相关性写成因果；
- 没有排除替代机制。

修正：

```text
1. 将“证明机制”改为“支持可能机制”。
2. 补充对照或机制验证实验。
3. 讨论替代解释。
4. 用证据强度匹配表达强度。
```

## 12.6 Results 与 Discussion 混淆

表现：

- Results 中大量解释文献和机制；
- Discussion 中重复图表趋势；
- 两部分内容高度重叠。

修正：

```text
1. Results 只写数据、趋势、差异和直接观察。
2. Discussion 写机制、文献比较、异常解释和边界。
3. 将文献比较移入 Discussion。
4. 将关键数值留在 Results。
```

## 12.7 图表不能自洽

表现：

- 图注不完整；
- 坐标轴单位缺失；
- 图中缩写未解释；
- 误差条无说明；
- 图和正文结论不一致。

修正：

```text
1. 补充完整图注。
2. 统一单位和符号。
3. 标明 n 和误差条。
4. 在正文中解释关键模式。
5. 删除不支持主线的图。
```

## 12.8 观察型研究过度因果化

表现：

```text
由于 X 增加，因此 Y 下降。
```

但数据只是相关观测。

修正：

```text
1. 改为“X 与 Y 下降相关”。
2. 说明可能机制。
3. 控制或讨论混杂因素。
4. 若无操控实验，避免强因果表达。
```

## 12.9 结论推广过大

表现：

- 实验室条件推广到真实环境；
- 单一物种推广到所有生物；
- 短期实验推广到长期稳定性；
- 单一区域推广到全球规律。

修正：

```text
1. 写明实验 / 观测条件。
2. 限定适用范围。
3. 将推广性表述改为假设或未来研究。
```

## 13. Agent 写作辅助策略

## 13.1 Agent 的总体任务

Agent 在辅助实验 / 观察型自然科学论文时，不应只润色语言，而应帮助用户完成以下工作：

```text
1. 识别科学问题；
2. 判断实验 / 观测是否围绕问题；
3. 重构证据链；
4. 检查变量、对照、重复、误差和统计；
5. 判断图表叙事是否清楚；
6. 区分结果、解释和推测；
7. 检查结论是否超过数据支持；
8. 将实验流水账改写为科学故事线。
```

## 13.2 从实验数据中提炼故事线

Agent 可使用以下流程。

### 第一步：列出全部证据

```text
图 1：……
图 2：……
图 3：……
表 1：……
补充图 S1：……
```

### 第二步：标注每个证据的功能

```text
确认研究对象
证明主要效应
验证变量关系
排除替代解释
支持机制
证明稳定性
说明边界条件
```

### 第三步：提炼中间发现

将每个实验从“做了什么”改写为“发现了什么”。

弱：

```text
进行了 XRD 测试。
```

强：

```text
XRD 结果表明，处理过程未破坏主体晶相。
```

弱：

```text
进行了不同温度实验。
```

强：

```text
温度升高在 60 °C 前促进反应，但超过该阈值后导致活性下降。
```

### 第四步：连接成证据链

```text
现象存在
→ 变量产生效应
→ 效应具有条件依赖
→ 对照排除替代解释
→ 表征支持机制
→ 稳定性或边界得到验证
```

### 第五步：形成主线句

```text
本文通过……证明……，进一步通过……排除……，并结合……支持……机制，从而说明……
```

## 13.3 Agent 检查结论是否超过数据支持范围

Agent 应逐句检查以下高风险词：

```text
证明
揭示了机制
决定
导致
普遍
显著优于所有
首次
完全
必然
广泛适用
```

检查规则：

```text
1. 是否有直接证据？
2. 是否有对照？
3. 是否有重复？
4. 是否排除替代解释？
5. 是否只在特定条件下成立？
6. 是否需要改为“支持”“提示”“可能”“在……条件下”？
```

### 结论降级示例

过强：

```text
These results prove that X controls Y.
```

更稳妥：

```text
These results support that X contributes to Y under the tested conditions.
```

过强：

```text
该机制适用于所有类似体系。
```

更稳妥：

```text
该机制可能适用于具有相似结构特征的体系，但仍需在更多材料中验证。
```

过强：

```text
X 导致区域生态系统退化。
```

更稳妥：

```text
X 与区域生态系统退化指标之间存在显著相关性，提示其可能是重要驱动因素之一。
```

## 13.4 Agent 诊断 Methods

Agent 应提出具体缺失信息，而不是泛泛说“方法不够详细”。

诊断模板：

```text
当前 Methods 已说明：
- ……

仍缺失：
- 样品来源：
- 对照设置：
- 重复数：
- 仪器参数：
- 数据处理：
- 统计检验：
- 异常值处理：
- 伦理 / 许可：
```

## 13.5 Agent 重构 Results

Agent 应将技术式标题改成发现式标题。

示例：

```text
原标题：3.1 SEM and TEM characterization
改写：3.1 X treatment produces a porous surface structure without disrupting the bulk morphology

原标题：3.2 Electrochemical performance
改写：3.2 The porous surface structure enhances current density under identical loading conditions

原标题：3.3 Mechanism analysis
改写：3.3 Intermediate stabilization explains the enhanced reaction rate
```

## 13.6 Agent 辅助写 Discussion

Agent 应把 Discussion 从“重复结果”提升为：

```text
1. 结果综合；
2. 机制解释；
3. 文献比较；
4. 异常处理；
5. 边界限定；
6. 未来研究。
```

可以要求用户提供：

```text
- 最核心的三张图；
- 与已有研究不同的结果；
- 目前最弱的机制证据；
- 是否有异常数据；
- 哪些条件没有测试。
```

## 13.7 Agent 润色原则

自然科学论文润色优先级：

```text
准确性 > 证据匹配 > 逻辑清晰 > 简洁性 > 风格流畅
```

润色时应避免：

- 把谨慎表达改得过强；
- 把观察结果改成因果判断；
- 删除条件限定；
- 增加作者没有证据支持的机制解释；
- 用华丽词汇替代准确描述。

## 14. 检查清单

## 14.1 全文逻辑检查清单

```markdown
## 科学问题
- [ ] 论文是否提出明确科学问题？
- [ ] 科学问题是否能通过实验 / 观测 / 测量回答？
- [ ] Introduction 是否说明该问题为何重要？
- [ ] 文献缺口是否具体到机制、证据、尺度、条件或方法？
- [ ] 结论是否回到该科学问题？

## 证据链
- [ ] 每个主要实验 / 观测是否有明确功能？
- [ ] 图表顺序是否形成发现链条？
- [ ] 每个结果小节是否有中间发现？
- [ ] 是否存在与主线无关的图表？
- [ ] 机制解释是否由多组证据支持？
- [ ] 是否处理替代解释？

## 实验 / 观测设计
- [ ] 自变量是否明确？
- [ ] 因变量是否明确？
- [ ] 控制变量是否明确？
```
